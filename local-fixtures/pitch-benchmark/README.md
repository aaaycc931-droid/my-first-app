# P111 本地音准基准工作区

此目录只提交协议说明和无真实数据的文本模板。真实录音、真实参与者明细、授权文件、标注表和本地基准结果不得提交 Git、上传聊天或打包进 APK。

使用步骤：

1. 把 `benchmark.example.json` 复制为 `benchmark.local.json`；后者已被 Git 忽略。
2. 使用随机样本 ID、匿名参与者令牌和独立授权记录令牌；不得填写姓名、电话、邮箱、学校、住址、精确位置或设备序列号。
3. 在运行任何引擎前确定 `tuning`、`development`、`acceptance-blind` 三组。最终盲测标签应由未参与调参的人封存；引擎和参数摘要必须在揭盲前冻结。
4. 同一份共同输入模板可以复制给多个引擎运行，但每个引擎必须使用独立 `benchmarkId`、`engineId`、版本和参数 SHA-256。
5. 只把引擎输出的 voiced、频率和处理时长写入 `result`。本工具不会读取音频，也不会进行网络请求。
6. 运行 `npm run validate:pitch-benchmark-protocol` 查看协议、P104 最低门槛、专业扩展门槛、总体和分层指标。

模板中的数据全部是占位示例，不是已采集证据，也不能用于产品准确度声明。
