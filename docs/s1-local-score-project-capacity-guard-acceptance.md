# S1 本机谱项目容量与资源保护切片验收

## 定位

本切片在既有本机谱项目保存、编辑、重开和播放闭环上增加确定、可复核的资源边界：

> 在同一 IndexedDB 原子事务内检查项目数量与序列化容量 → 未超限才写入 → 失败时保留最后已保存 revision → 条件恢复后允许用户重试。

它是 S1 本机数据保护 implementation candidate，不代表云同步、账号、导入导出、自动清理、付费扩容、教育审核或推广测试已经完成。

## 应用设定限制

- 最多保存 50 个本机谱项目。
- 全部本机谱项目按 UTF-8 JSON 确定性计算，合计最多 5 MiB。
- 更新既有项目时，以“当前全部记录大小 - 旧记录大小 + 新记录大小”计算。
- 正好达到限制允许提交；超过任一限制时拒绝整个事务。
- 损坏或未来版本记录继续保留，并纳入数量与容量计算；应用不会为腾出空间而自动删除、覆盖、压缩或修改任何记录。

## 失败分类与恢复

- `capacity`：应用自身的 50 个 / 5 MiB 限制。
- `quota`：浏览器或 Android WebView 分配给 IndexedDB 的 quota。
- `write-failed`：普通本机存储写入失败。
- `transaction-failed`：IndexedDB 事务失败或中止。
- `conflict`：预期 revision 与当前已保存 revision 不一致。

每类失败均使用简体中文说明来源和“原有项目保持不变”。创建或编辑 proposal 只有在事务完成后才发布到界面；失败后按钮恢复可用，用户清理空间或恢复存储条件后可原样重试。

## 自动验收

1. 低于限制与正好达到限制均可保存；超过限制拒绝。
2. 达到项目数量上限时拒绝新项目，已有项目保持不变。
3. 编辑使总容量超限时不发布新 revision，IndexedDB 仍保留旧 revision。
4. IndexedDB quota、普通写入失败、事务失败/中止使用不同状态与中文提示。
5. 失败事务不留下部分记录。
6. 放宽容量或恢复事务条件后，同一创建或编辑可重试成功。
7. Quality 覆盖全部 `test:*` 命令且每项恰好执行一次。

## 人工 QA 状态

- 桌面浏览器 QA：`NOT_EXECUTED`
- Android WebView QA：`NOT_EXECUTED`
- Android 真机 QA：`NOT_EXECUTED`

人工 QA 应分别验证：接近上限、正好达到、超过上限、quota、事务中止、失败后刷新/重开仍为旧 revision，以及恢复条件后的重试。

## 明确未完成

- 云同步、账号系统、导入导出；
- 自动清理、自动压缩、静默改写或付费扩容；
- 与容量保护无关的编辑器 UI 重构；
- 浏览器、Android WebView 或物理设备的真实 quota 数值与回收行为证据；
- P119d 逐题六维双教师审核、教育有效性、正式评分或推广测试批准。

P119d 已批准的 153 项候选批次继续冻结。两份独立逐题六维审核与教育有效性仍为 `NOT_EXECUTED`，正式评估门禁仍为 `BLOCKED`；本切片不改变这些状态。
