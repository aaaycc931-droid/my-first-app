# P116d 节奏听写验收

最后更新：2026-07-21

P116d 是 P116 节奏专业套件的第四个最小完整纵向切片。它复用版本化本地节奏题库、统一 Activity `staff-notation` 输入、`score-document-v1` 确认修订、本机复练与学习画像，提供“完整听题—编辑节奏谱面草稿—预览—检查—确认—事件对照”的离线闭环。

## 用户闭环

1. Android 扩展节奏入口只显示 4/4、BPM 与难度，不提前显示目标事件。
2. 用户主动完整播放一小节本地合成节奏；播放完成前谱面网格保持锁定。
3. 用户在固定拍内网格中添加、移除击拍事件，并随时查看当前草稿预览。
4. 用户先执行结构检查；任何后续修改都会让旧检查失效。
5. 用户明确确认后，系统把当前草稿冻结为 session-only、local-only 的 `score-document-v1` 节奏修订，并通过统一 Activity `staff-notation` 答案提交。
6. 只有确认修订提交后才揭示版本化目标，显示漏记与多记的具体拍内位置；一致时也只表达事件一致性。
7. 随机扩展节奏页一次只挂载一种活动；切换活动会停止旧播放、清除旧作答并换题，视读／找错的可见目标不能泄露听写答案。固定复练题不提供专业活动切换，避免把已知目标复用于隐藏活动。

## 草稿与输入不变量

- 基础与进阶提供四分／八分位置；挑战提供四分、八分、十六分与三连音位置，固定网格包含当前版本化题库全部目标事件。
- 草稿事件必须唯一、位于当前四拍小节且属于当前难度允许网格；空草稿不能通过检查。
- 检查只验证结构和可表示性，不读取标准答案，也不生成正确率。
- 确认要求当前草稿指纹与最近一次检查一致；编辑、删除或清空会立即移除旧检查指纹。
- 确认文档保存内容指纹、题目身份、事件拍点、4/4 拍号与 `confirmed` 状态；不保存 attempt 元数据、音频、用户身份或评分字段。
- `score-document-v1` 使用 `documentKind` 区分既有音高谱面与节奏谱面。节奏文档按题目与规范化事件内容寻址；不同 attempt 产生同一内容时必须得到完全相同的不可变文档对象，不能出现相同 `documentId + revision` 对应不同来源内容。

## Fail-closed 与隐私边界

- 播放 token 绑定当前题目；重播、手动停止、全局停止、后台、重置与卸载都会清除完整播放资格、草稿、旧确认、迟到计时器和脏 attempt。播放期间任何 AudioContext 状态中断都会立即作废；时间线未完整走完时也拒绝授予听题资格。
- 迟到播放完成回调不能重新解锁已作废题目；本地音频准备失败时显示中文恢复原因。
- 不使用麦克风，不上传或保存声音；不新增账户、云端、数据库、存储 schema 或依赖。
- 不显示总分、百分比、等级、通过／失败，不从事件对照推断用户听辨、演奏或教育能力。
- 本机复练只沿用现有稳定节奏题目标识，不保存具体草稿、选择或事件差异。

## 自动验收

- focused domain tests 覆盖网格、空草稿拒绝、检查指纹、修改失效、确认冻结、伪造／跨题／陈旧 attempt 拒绝、挑战三连音表示、正确与差异事件对照及非评分字段边界。
- 真实挂载 React 行为测试覆盖完整播放前锁定、草稿编辑、预览、检查、确认、Activity 提交、事件揭示、修改失效、全局停止、AudioContext 中断与迟到计时器。
- Android App 集成行为覆盖单活动挂载，以及听写确认前不出现视读／找错目标和题目事件对照。
- Android validator 检查领域模型、Activity `rhythm-dictation` / `staff-notation`、确认文档、移动端入口、生命周期 token 与 bundle 文案。
- GitHub `quality` 必须运行 focused 与挂载行为测试；`android-local` 继续负责同步、原生单测、Debug APK 构建和独立工件校验。

## 未声明完成

- 自动测试不等于真实 Android 扬声器、触控、后台、音频焦点或长时间稳定性证据。
- 当前网格是原创的拍内事件编辑器，不是完整五线谱排版、正式制谱、正式转写或 MusicXML round-trip。
- 当前题库和解释尚未完成两名独立视唱练耳教师审核，也不证明教育有效性。
- 麦克风拍手、MIDI 输入、正式节奏评分、跨设备同步和云端题库不属于本切片。

QA level recommendation：**strict**。
