# P43 — 视唱练耳系统功能细节设计标准与功能地图

> 规范关系：本文件保留功能语义和产品边界；其中历史“当前 MVP”状态不再作为生产事实。当前事实见 `docs/mvp-status.md`，正式版 V1 范围和退出条件见 `docs/final-release-definition-of-done.md`，实时差距见 `docs/final-release-status-matrix.md`。

## 1. 最终产品定位

本产品最终定位为：**面向中文用户的视唱练耳练习系统**。

它以音高、节奏、听辨、视唱、练习反馈为主体；以五线谱识别、系统内置练习、本地旋律参考音频、手动创建练习和未来私人歌曲导入作为练习内容来源；通过草稿检查、临时目标、非评分反馈和未来正式评分，帮助用户完成持续训练。

五线谱识别不是单独的 OCR 工具，而是为视唱练耳提供练习内容和目标草稿的核心输入能力。私人歌曲练习不是当前最大主体，而是视唱练耳系统的补全功能和未来个性化练习素材来源。练习反馈是训练闭环的关键支撑，不是边缘功能。

## 2. 功能设计总原则

所有自动生成、自动识别、自动分析出来的结果，都不能直接当成最终结果。

必须经过：

```text
预览
→ 检查
→ 修改 / 确认
→ 再进入练习
```

适用对象包括：

- 五线谱识别结果；
- 目标音高曲线草稿；
- 私人歌曲分析结果；
- 节奏分析结果；
- 分句分析结果；
- 练习目标生成结果；
- 未来正式评分目标。

默认规则：自动结果是草稿，不是最终目标，不是正式转写，不应直接评分。

## 3. 历史 P43 MVP 边界（已过期）

当前 MVP 仍保持：

- browser-local；
- session-only；
- no account；
- no database；
- no cloud；
- no upload；
- non-scoring；
- no final target；
- no official transcription；
- no source separation；
- no full-song extraction；
- no persistent library。

本阶段是 docs-only，不修改 runtime，不新增 UI，不新增账号、数据库、云端、上传、评分、final target、correction editor 或 dependency；不修改 `/api/recognize`、parser / converter 或 piano runtime。

## 4. 主体功能与附属功能表

### 4.1 练习内容生成与导入

- **功能作用**：为视唱练耳练习提供目标素材和目标草稿。
- **用户最终能做什么**：从五线谱、系统内置练习、本地旋律参考音频、手动创建内容和未来私人歌曲中生成可练习目标。
- **附属功能**：五线谱识别、系统内置练习库、本地旋律参考音频导入、手动创建练习、目标音高曲线草稿、节奏目标草稿、分句草稿、未来私人歌曲导入。
- **当前状态**：已有 `/practice` 功能导航、临时检查草稿练习目标、目标音高曲线草稿、节拍器、节奏练习、起音检测等基础能力；仍是 browser-local / session-only / non-scoring。
- **未来方向**：把五线谱识别升级为“乐谱到练习目标”的输入系统；补充内置练习；逐步加入手动创建、私人歌曲导入和可保存的练习库。
- **使用细节要求**：任何自动生成目标必须先展示草稿来源、范围、置信度或限制，再允许用户检查、修改或确认；未经确认不得进入正式评分。

### 4.2 音高练习

- **功能作用**：训练用户唱准单音、音程和旋律。
- **用户最终能做什么**：听参考音、录制自己的声音、查看偏高 / 偏低 / 接近 / 无可靠音高反馈，并复练。
- **附属功能**：单音模唱、音程模唱、旋律模唱、目标音高曲线、本地音高估计、参考音播放、录音、音高稳定性分析、复练、未来正式音准评分。
- **当前状态**：已有本地音高估计、录音和目标音高曲线草稿基础；反馈仍是非评分诊断。
- **未来方向**：补充标准化单音 / 音程 / 旋律题库，形成正式音准评分前的检查与确认链路。
- **使用细节要求**：反馈必须区分偏高、偏低、接近、无可靠音高；显示音高稳定性；提示用户是否需要重录或复练；不得把低置信度音高当成正式结果。

### 4.3 节奏练习

- **功能作用**：训练用户稳定跟拍、识别节奏和准确发声 / 点击时机。
- **用户最终能做什么**：跟随节拍器和 count-in 练习，进行 tap rhythm practice，查看 early / late / close / missed / extra 等非评分反馈。
- **附属功能**：节拍器、count-in、subdivision、tap rhythm practice、latency calibration、audio onset detection、onset-to-rhythm feedback、节奏时间线、节奏稳定性、未来正式节奏评分。
- **当前状态**：已有节拍器、节奏练习、tap latency calibration、audio onset detection、onset-to-rhythm feedback 和节奏时间线诊断基础。
- **未来方向**：把节奏反馈从诊断扩展为可检查、可复练、可保存的训练闭环，再进入正式节奏评分研究。
- **使用细节要求**：必须说明 latency 与 alignment 会影响反馈；early / late / close / missed / extra 是练习反馈，不是正式评分；需要显示节奏时间线和稳定性趋势。

### 4.4 视唱练习

- **功能作用**：把音高、节奏和谱面阅读整合为完整视唱训练。
- **用户最终能做什么**：从乐谱或练习目标开始，听 count-in，录音演唱，查看音高与节奏反馈，按分句复练。
- **附属功能**：从五线谱识别生成视唱目标、从内置练习生成视唱目标、从本地旋律草稿生成临时目标、count-in、录音、音高反馈、节奏反馈、分句练习、复练、未来正式评分。
- **当前状态**：已有 `/practice` 练习导航和非评分反馈基础，尚未形成正式视唱评分。
- **未来方向**：建立“目标检查 → 进入视唱 → 音高 / 节奏反馈 → 分句复练 → 长期进步”的主流程。
- **使用细节要求**：视唱目标必须有来源说明；来自识别或分析的目标必须先检查；练习反馈要分开显示音高和节奏问题，避免直接给出正式成绩。

### 4.5 听辨练习

- **功能作用**：训练用户听出单音、音程、节奏和旋律。
- **用户最终能做什么**：听题目、输入答案、查看错误解释、调整难度并复练错题。
- **附属功能**：单音听辨、音程听辨、节奏听辨、旋律听写、题目播放、答案输入、错误解释、难度调整、错题复练。
- **当前状态**：Android 本地入口已提供单音、音程、节奏和三音旋律听写，每类均有基础、进阶、挑战三档且每档至少 20 个稳定版本化组合；完整一轮随机题序不重复。最多 12 项的本机错题复练队列支持答错置顶、答对移除和二次确认清除；稳定题目标识与旧 catalog v1 原地迁移避免题库扩容后旧题漂移。真实挂载 React App 与纯逻辑自动回归覆盖该闭环、三难度题量、组合唯一性、浏览历史、迁移和存储异常。该队列只保存最小题目复现目标，不是学习历史或评分；共享 Web `/practice` 与课程练习继续使用旧题库且不持久化到该队列；自动化不等于真机或教育审核。
- **未来方向**：先完成 Android 真机跨重启、存储异常、离线复练和三难度听感验证；长期练习记录、弱项分析与正式难度自适应需独立设计和校准。
- **使用细节要求**：题目播放、答案输入和错误解释必须清晰；本机复练存储失败不得阻断练习；不得把复练次数或队列状态当作成绩、准确率或通过/失败判断。难度调整早期保持手动选择，未校准前不得冒充基于表现的正式自适应。

### 4.5A 钢琴辅助

- **功能作用**：为听辨与视唱提供不联网的本地找音工具。
- **用户最终能做什么**：在 Android 独立“本地参考钢琴”页或四类练习内展开同一钢琴，切换 C3–C4、C4–C5、C5–C6 音域，用多指、音量和延音播放本地合成参考音，并随时停止全部声音。
- **附属功能**：13 键半音键盘、中央 C 标识、键盘 Enter/空格、8 声部上限、音频失败重试、10 秒残音看门狗、导航/后台/卸载清理。
- **当前状态**：P94 自动模型与 React 行为测试已覆盖；共享 Web 和课程入口默认不显示，Android 显式开启。尚未完成真实手机触控、扬声器、横竖屏和 System WebView QA。
- **未来方向**：先完成 Android 真机验证；MIDI、录音、88 键、采样音色、力度、伴奏与钢琴课程均延期。
- **使用细节要求**：只能称本地合成参考音，不声称真实钢琴音色、零延迟或精准校音；弹奏不保存、不上传、不生成成绩或正式评分；题目音与钢琴音必须互斥，所有离开/中断路径必须停止声音。

### 4.6 练习反馈系统

- **功能作用**：形成视唱练耳训练闭环，告诉用户哪里接近、哪里需要复练。
- **用户最终能做什么**：查看音高、节奏、起音、稳定性、分句反馈和练习建议，并在未来查看长期进步。
- **附属功能**：音高反馈、节奏反馈、起音反馈、稳定性反馈、分句反馈、练习建议、长期进步反馈、未来正式评分。
- **当前状态**：已有非评分音高 / 起音 / 节奏诊断基础；Android 已提供用户主动开启的实时曲线、会话录音回放、版本化练声目标、目标音块叠加、手动分段复练和接近 / 偏高 / 偏低 / 不可靠反馈。用户可主动保存本机曲线与可选录音；默认不保存、不上传。仍不产生分数、等级、通过 / 失败或正式评估。
- **未来方向**：进入 P103 高级观察诊断，并以真实人声、设备与教育审核校准稳定度、颤音和尾音趋势；达标前不讨论正式评分。
- **使用细节要求**：反馈必须可解释，不只显示数字；低置信度、无可靠音高、漏检和额外起音要明确提示；练习建议应优先服务复练。

### 4.7 练习目标检查与修正

- **功能作用**：保证草稿目标进入练习前被用户理解、检查和确认。
- **用户最终能做什么**：预览草稿、检查 voiced coverage、first / last voiced frame、full draft、voiced span 和 manual range，创建临时练习目标，并在未来进行 pitch correction、phrase correction 和 final approval。
- **附属功能**：草稿预览、voiced coverage、first / last voiced frame、full draft、voiced span、manual range、临时练习目标、未来 pitch correction editor、未来 phrase correction、未来 final approval。
- **当前状态**：已有临时检查草稿练习目标和目标音高曲线草稿，仍不是 final target。
- **未来方向**：增加明确的修正编辑器和最终确认步骤，使识别 / 分析草稿可以变成可信练习目标。
- **使用细节要求**：必须展示草稿覆盖范围和可靠性；用户修改或确认前不得把草稿当作正式目标；临时目标只在当前 session 内有效。

### 4.8 练习记录与进步追踪

- **功能作用**：让用户看到长期训练变化，支持持续练习。
- **用户最终能做什么**：查看历史练习、复练薄弱题型、观察音高稳定性和节奏稳定性变化。
- **附属功能**：练习历史、错题 / 弱项复练、长期音高趋势、长期节奏趋势、目标完成记录、未来多设备同步。
- **当前状态**：Web 已有私人 practice attempts 与课程答案摘要；Android 有最多 12 项的本机错题复练队列，以及用户主动保存、可回看/删除/清空/导出的本机练声记录。它们尚未形成统一长期趋势、正式技能画像或跨设备同步。
- **未来方向**：先完成 Android IndexedDB、跨重启、升级与配额真机验证，再把 Web 私有记录整理为可见历史和解释型复练建议。
- **使用细节要求**：记录必须尊重用户数据所有权；提供删除和导出；不得把记录系统扩展成社区排行或公开主页。

### 4.9 私人歌曲练习补全

- **功能作用**：作为未来个性化练习素材来源，补全视唱练耳系统。
- **用户最终能做什么**：导入私人歌曲，生成歌曲练习目标草稿，检查修正后进入视唱练耳练习。
- **附属功能**：私人歌曲导入、私有云分析、source separation、vocal melody extraction、beat / phrase analysis、歌曲练习目标草稿、用户检查与修正、私人曲库、进入视唱练耳练习。
- **当前状态**：当前不实现上传、云端、账号、私有曲库、source separation 或 full-song extraction。
- **未来方向**：在账号、隐私、版权、云处理和数据删除设计完成后，作为私人练习素材管线推进。
- **使用细节要求**：私人歌曲分析结果必须是草稿；需要用户检查与修正；只能服务私人练习，不做公开播放、公开分享或资源交换。

### 4.10 账号与私人数据系统

- **功能作用**：未来支撑私人曲库、私有云分析、练习记录和多设备同步。
- **用户最终能做什么**：管理私人曲库、保存练习记录、同步多设备、使用会员 / 权限 / 额度，并删除或导出个人数据。
- **附属功能**：登录、私人曲库、私有云分析、练习记录、多设备同步、会员 / 权限 / 额度、数据删除、数据导出。
- **当前状态**：当前不实现账号、登录 UI、auth API、数据库或会员系统。
- **未来方向**：单独进行账号 / 隐私 / 安全 / 权限 / 数据生命周期设计。
- **使用细节要求**：账号不服务于社区、公开主页、互相关注、评论、私信、公开上传、公开分享或公开资源库。

## 5. 五线谱识别细节设计：从 OCR 工具升级为练习目标输入系统

五线谱识别不是单独 OCR 工具，而是视唱练耳练习内容生成入口。未来完整流程应该是：

```text
上传乐谱图片 / PDF
→ 谱面预览
→ 选择识别范围
→ 识别音符、节奏、小节、拍号、调号
→ 生成识别草稿
→ 标出不确定区域
→ 用户检查与修改
→ 小节时值 / 拍号 / 调号 / 音高范围校验
→ 生成视唱练习目标或节奏练习目标
→ 进入练习模式
```

未来需要的细节功能包括：

- 图片上传；
- PDF 上传；
- 页码选择；
- 谱面预览；
- 缩放；
- 拖动；
- 旋转；
- 裁剪；
- 选择识别范围；
- 整页 / 某一行 / 某几个小节识别；
- 音符识别；
- 休止符识别；
- 节奏识别；
- 拍号识别；
- 调号识别；
- 小节线识别；
- 识别置信度 / 不确定提示；
- 原谱与识别结果对照；
- 手动修改音高；
- 手动修改时值；
- 手动修改休止符；
- 手动修改拍号；
- 手动修改调号；
- 小节时值校验；
- 转换为视唱练习目标；
- 转换为节奏练习目标；
- 生成目标音高曲线；
- 进入非评分练习反馈；
- 未来再进入正式评分。

识谱结果默认是草稿，需要检查，不是最终目标，不是正式转写，不应直接评分。任何“乐谱到练习目标”的路径都必须先完成预览、检查、修改 / 确认，再进入练习。

## 6. 明确不做的功能

本产品不做：

- 社区；
- 公开主页；
- 用户互相关注；
- 评论区；
- 私信；
- 公开上传；
- 公开音乐分享；
- 公开资源库；
- 用户上传版权音乐的公开播放；
- 用户之间资源交换。

## 7. 未来增强方向

- 内置视唱练耳题库和分级练习路径。
- 乐谱识别后的草稿编辑、校验和目标生成。
- 更完整的 pitch correction editor、phrase correction 和 final approval。
- 可解释的正式音准评分、节奏评分和综合视唱评分。
- 账号、私人曲库、私有云分析、练习记录同步和数据删除 / 导出。
- 私人歌曲练习补全，但只作为私人练习素材来源，不扩展为社区或公开资源平台。
