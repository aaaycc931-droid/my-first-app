# S1 本机谱项目第二个编辑切片验收

## 定位

本切片在首个“新建 → 输入 → 保存 → 播放 → 撤销/重做 → 再打开”闭环上继续交付：

> 追加第二小节 → 选择目标小节 → 输入或选择已有事件 → 修改并保存 → 图形五线谱检查 → 安全删除末尾空小节。

它仍是本机 S1 implementation candidate，不代表完整 S1、正式制谱器、教育审核或推广测试已经完成。

## 本批范围

- 第一 part / staff / voice 可连续追加一个编号唯一、无间隙的空小节。
- 用户可选择目标小节并添加基础音符或四分休止符。
- 事件列表和图形五线谱均保留精确的 part、staff、voice、measure 位置。
- 点击列表或图形 token 可选择已保存事件，表单预填后通过现有版本化命令更新。
- 只允许删除最后一个空小节；唯一小节或非空末尾小节必须失败关闭。
- 小节和事件修改继续遵守 save-first、连续 revision、stale writer 冲突、撤销/重做和重新打开语义。
- 第一声部提供只读 SVG 五线谱检查视图，覆盖 C4–C5、二分/四分/八分音符、四分休止符、拍号和小节线。
- 图形视图只读取当前已保存 revision；超拍时失败关闭，不修改或删除隐藏声部。

## 自动验收

1. 领域测试证明连续追加、无重复/间隙、stale revision 拒绝、唯一/非空小节拒删、撤销/重做和序列化恢复。
2. 面板行为测试证明第二小节选择、写入、事件更新、非空拒删、清空后删除和撤销恢复。
3. 图形适配器和组件测试证明确定性布局、三种拍号、超拍阻断、隐藏声部隔离、完整 aria、点击与键盘选择。
4. IndexedDB 恢复测试证明真实事务下的重开、撤销/重做、并发 CAS、损坏/未来版本共存、删除和事务取消恢复。
5. Quality 必须覆盖全部 `test:*` 脚本且每项恰好一次；lint、typecheck、生产构建和 Android 本地构建继续通过。

## 人工 QA

QA level recommendation：**standard**。本切片新增 UI 选择、目标小节和显示路径，但不改变音频播放语义。

在真实浏览器和 Android WebView 上分别执行：

1. 新建项目，追加第二小节，分别向两小节写入事件。
2. 从图形 token 和列表各选择一次事件，修改音高或时值并保存，确认蓝框、列表和重新打开后的内容一致。
3. 非空末尾小节点击删除应显示拒绝原因；删除其事件后才能删除该小节。
4. 对小节和事件修改执行撤销/重做、刷新或重启后重新打开，确认 revision、内容和历史一致。
5. 模拟 stale writer 与存储失败，确认未保存 proposal 不进入图形视图。
6. 横向滚动多小节预览，并用键盘 Enter / Space 选择 token。

本仓库没有浏览器或 Android 真机执行记录时，上述人工 QA 必须保持 `NOT_EXECUTED`。

## 明确未完成

- 多 part / staff / voice 编辑，跨小节移动、复制/粘贴或拖动；
- 调号、速度、附点、连音、歌词、和弦、指法、简谱或完整排版；
- 与音频调度同步的真实播放光标、节拍器和选择区循环；
- 空间配额恢复 UI、版本升级迁移 UI、导入导出、打印和云同步；
- 浏览器或 Android WebView 手动 QA、Android 三档真机证据；
- P119d 逐题双教师审核、教育有效性、正式评分或推广测试批准。

P119d 已批准的 153 项批次继续冻结；逐题六维双教师审核仍按产品所有者决定延期到正式推广测试启动时，本切片不会改变该门禁。
