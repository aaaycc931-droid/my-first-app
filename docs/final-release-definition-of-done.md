## 2. 正式版 V1 产品定义

正式版 V1 面向中文学习者，当前以可私下分发的 Android 本地 APK 为优先载体，同时保留 Web 私有学习平台。APK 的核心练习必须在无生产网站、无账号和无网络时可用；Web 用户能够登录，使用系统课程或自己的私有素材完成听、唱、拍、读谱和钢琴辅助练习，获得可解释的非评分反馈，保存进度、复练薄弱点，并导出或删除自己的数据。

正式版 V1 必须保持以下产品边界：

- 用户可见主流程使用简体中文。
- 私有数据只有所有者可访问。
- 自动识别、自动生成和自动分析的结果必须经过“预览 → 检查 → 修改 / 确认 → 再进入练习”。
- 没有达到正式评分门槛时，只显示诊断、答案说明和练习反馈，不显示考试分数、等级或通过/失败。
- 不提供社区、公开个人页、关注、评论、私信、公开上传、公开曲库或用户间资源交换。

### 3.2 条件能力

下列能力只有在对应门槛全部通过时才可进入 V1；否则必须继续使用非评分或降级路径，且不阻塞 V1 发布：

- 正式音准、节奏或综合考试评分；
- AI 生成的解释或学习建议；
- AI 音乐伙伴、角色成长、工具型智能体和云端长期记忆；
- 第三方登录；
- MIDI 硬件输入。

AI 只能解释已经结构化、版本化的测量结果，不能替代音高、节奏或权限判断。

### 3.3 V1 明确不做

- 社区、公开主页、关注、评论、私信；
- 公开上传、公开音乐分享、公开曲库、用户间资源交换；
- 支付、会员收费和复杂商业化；
- 原生 iOS App；
- 大型在线歌曲库；
- 高级 MIDI 录制/编辑；
- 未经过基准与教育审核的正式考试分数；
- 未经过版权、隐私和删除设计的整曲 source separation 或公开播放。

这些能力只能通过明确的后续版本决策加入，不能在开发过程中静默扩大 V1。

## 4. 功能完成门槛

每个必须功能只有同时满足以下条件才可标记为 `PASS`：

1. 存在真实运行时实现，不依赖未标明的 mock。
2. 用户第一步、空状态、加载状态、成功状态、错误状态和恢复动作完整。
3. reset、clear、replace、stale 和重复提交行为有定义并经过测试。
4. 所有自动结果明确标记来源、版本、置信度和草稿/正式边界。
5. 适用的数据有所有者、RLS、删除语义和版本字段。
6. 关键路径有 focused tests、类型检查、生产构建和浏览器交互证据。
7. 用户可见文案为简体中文，不使用误导性的评分、正式转写或最终目标表述。
8. 对应文档、迁移、运行手册和回滚方式已经更新。

## 5. 课程与内容门槛

- 至少有一条从零开始可完成的基础课程路径，覆盖单音、音程、节奏、旋律听写和基础视唱。
- 每个核心题型至少提供 3 个明确难度层级；每层至少 20 个版本化题目或等价的可验证生成组合。
- 每道题保存目标版本、答案/规则来源、解释和适用难度。
- 课程顺序、题目正确性、中文解释和难度递进由至少 2 名具备视唱练耳教学经验的审核者复核。
- 抽样复核题目正确率必须为 100%；任何已知错误题在修复前不得进入发布题库。
- 用户能够从错误答案或薄弱点进入明确的复练路径，不以黑箱排名替代解释。
- Android 本机复练的加入、移除、清除确认、导航恢复和存储失败降级必须有真实挂载移动 React 入口的行为回归；源码字符串检查不能单独作为该闭环完成证据。
- Android 钢琴辅助同时提供独立页和核心练习内嵌入口；至少覆盖一个完整半音音域、音域切换、多指、音量、延音、全停、中文失败重试和后台/导航/卸载清理。弹奏不得保存、上传或冒充评分，并须有纯状态与真实挂载 React 行为回归。
- 上一条只是钢琴最小安全闭环，不构成专业完成。V1 专业钢琴还必须满足 `docs/professional-competitive-roadmap-perfect-piano-earmaster.md` 的 88 键、采样音色、复音、完整移动交互、录制/节拍器/循环、音色许可、低延迟和真机门槛；当前三角波只能作为显式兼容降级。
- 视唱练耳活动范围至少覆盖单音/音程、和弦/转位/进行、音阶/调式、节奏视读/回模/找错/听写、旋律听写/回唱/视唱和声乐练习；每个活动必须有版本化目标、答案模式、中文解释、三档难度、复练入口和教育审核，不得用四个选择题面板宣称课程完整。

## 6. 算法与反馈门槛

### 6.1 数据集

- 音高基准至少包含 200 个合成/乐器单音片段和 100 个真实人声片段，覆盖不少于 20 位说唱者/歌唱者、常见男女声区、强弱音、颤音、背景噪声和至少 4 类真实设备。
- 节奏基准至少包含 200 个目标序列和 100 个真实点击/人声/乐器起音片段，覆盖均匀、切分、休止、密集、漏拍、多拍和至少 4 类真实设备。
- 数据集记录授权、来源、设备、采样率、标注者、标注版本和排除原因；训练/调参样本与最终验收样本隔离。

### 6.2 非评分反馈

- 清洁单音样本的音高绝对误差中位数不高于 25 cents，P95 不高于 60 cents，八度错误率不高于 1%。
- 真实人声样本的音高绝对误差中位数不高于 35 cents，P95 不高于 100 cents，八度错误率不高于 2%。
- 延迟校准后的节奏匹配绝对误差中位数不高于 60 ms，P95 不高于 150 ms。
- 单调对齐、目标/输入顺序和 miss/extra 守恒性质在确定性回归集中 100% 通过。
- 低置信度、无声、复音、强噪声和越界输入必须降级为“不足以判断”，不得产生强结论。
- 每个结果包含算法版本、目标版本、置信度、适用条件和可解释证据。
- 实时监听和录音提交后分析必须使用不同的版本标识与验收报告。录音提交后分析须重新解码本地 PCM，完成 voiced/unvoiced、连续音高、八度错误抑制、音符/片段分段和目标对齐；不得只重用实时帧缓存冒充专业离线分析。
- 录音后处理需分层报告 voiced/unvoiced F1、Raw Pitch Accuracy、Raw Chroma Accuracy、cents 中位数/P95、八度错误率、拒答率和三档 Android 处理实时系数；目标先验不得把错误演唱强行修正成正确音。

### 6.3 正式评分附加门槛

正式分数必须另有独立审批，至少包括教育专家审核、设备分层公平性、置信度校准、评分重复性、误判分析和版本迁移方案。未全部通过时，正式版 V1 仍保持非评分反馈。

## 7. 性能与可靠性门槛

- 发布候选窗口连续 14 天，核心 Web/API 可用性不低于 99.5%。
- 核心页面真实用户 P75：LCP 不高于 2.5 秒、INP 不高于 200 ms、CLS 不高于 0.1。
- 本地按钮到 UI 状态变化 P95 不高于 100 ms；用户手势到 Web Audio 计划启动 P95 不高于 150 ms。声卡/蓝牙物理输出延迟单独记录，不冒充应用延迟。
- 连续完成 20 轮播放、录音、重置和切换后，不得残留活动媒体轨、失控计时器或持续增长的可复现内存泄漏。
- 一页乐谱的异步识别 P95 不高于 5 分钟；5 分钟以内音频的异步分析 P95 不高于 10 分钟。超时必须显示状态、取消和重试。
- 发布候选期间核心用户会话无未处理应用错误比例不低于 99%。
- 所有性能数字必须记录设备、浏览器、网络、样本量和测试日期。
- Android 在飞行模式下冷启动到可操作本地首页 P95 不高于 3 秒，不显示远程错误、登录阻塞或旧的私人数据。

## 8. 安全、隐私与数据门槛

- 每个用户拥有的数据表都有 RLS 所有权策略以及匿名、本人、跨用户测试。
- 浏览器不能直接写敏感练习、评测、任务或私有资产表；写入通过最小权限 RPC/API。
- service role、数据库密码、邮件和 worker 密钥只存在于受保护的服务端 Secret，不进入客户端 bundle、日志、截图或仓库。
- 私有对象 URL 默认有效期不超过 10 分钟；过期、换用户和删除后访问失败。
- 用户发起删除后立即撤销产品访问，活动数据库/对象/派生产物在 24 小时内删除；备份中的残留不超过已公开的备份保留期。
- 用户数据导出在 24 小时内完成，包含结构化数据、资产清单、版本和生成时间。
- 生产数据库至少每日备份；RPO 不高于 24 小时，RTO 不高于 4 小时；正式发布前完成一次有证据的恢复演练。
- 依赖扫描不得存在未接受的 Critical/High 漏洞；认证、上传、任务和高成本接口有速率限制与滥用恢复方式。
- 有公开的隐私说明、数据用途、保留期、删除入口、第三方处理方清单和版权/用户素材边界。
- 如果产品允许未成年人使用，必须在发布前确定年龄范围、监护人/同意要求、默认隐私设置和未成年人数据删除流程。

## 9. 可访问性与兼容性门槛

- 项目目标为 WCAG 2.2 AA；核心路径无已知阻断级可访问性问题。
- 登录、课程、练习、上传/导入、草稿确认、历史、导出和删除可以仅用键盘完成。
- 焦点顺序、可见焦点、表单标签、错误关联、对比度、缩放和屏幕阅读器名称经过检查。
- 最小支持宽度为 360 CSS px；360、390、768、1024 和 1280 px 均无阻断操作的溢出或遮挡。
- 浏览器矩阵覆盖桌面 Chrome/Edge/Firefox/Safari 和 iOS Safari、Android Chrome 的当前及前一主要版本；Android APK 还必须覆盖当前及前一主要 Android System WebView。
- 正式发布前至少在 2 台真实 iOS 设备和 3 台不同性能档位的真实 Android 设备完成音频播放、麦克风、录音、回放和核心练习；3 台 Android 同时执行已安装 APK 路径。
- 不支持的浏览器、麦克风、MIDI 或音频能力必须提供中文原因和可继续使用的降级路径。

### 9.1 Android APK 专项门槛

- 固定应用 ID 为 `com.aaaycc931.solfeggio`；版本号、versionCode 和变更记录可追溯。
- Capacitor 只加载 APK 内的静态 `webDir`，不得配置远程 `server.url`；核心练习 bundle 不包含生产 Supabase URL、key 或会话依赖。
- 本地私测/正式 APK 不声明 `android.permission.INTERNET`；后续新增网络能力必须由产品所有者重新批准并单独做隐私、安全和断网设计。
- 产生可直接安装的签名 APK；文件保存 SHA-256 校验和、签名证书指纹、构建 commit 和依赖审计结果。AAB、Play 和其他商店工件不属于当前范围。
- 应用切到后台、停止、切换练习和 WebView 销毁时释放音源、录音轨与计时器；恢复时不自动播放或录音。
- 正式签名材料只存在于受保护 Secret 或安全密钥存储，不进入仓库、公开工件、日志或截图；调试签名包不得冒充正式发布包。
- 从上一私测/正式版本原地升级后，本地设置和权限状态符合预期；回滚包和恢复步骤已经演练。

## 10. 可运维性与发布门槛

- 生产错误追踪、结构化日志、关键任务指标、告警接收人和隐私过滤已配置。
- 有认证故障、数据库不可用、存储失败、worker 堆积、第三方识别失败和错误发布的运行手册。
- 数据库迁移可重复或有明确前置条件，完成 dry-run、生产备份、执行记录和回滚/前滚方案。
- 每个发布使用受保护分支、审查过的 PR、自动 CI 和版本化变更记录。
- CI 必须通过 lint、typecheck、build、focused tests、依赖审计和适用的 schema/RLS 检查。
- Android 发布 CI 还必须验证本地 bundle、无远程运行时配置、最终权限清单、APK 构建、签名、校验和与安装包结构。
- 生产发布后完成核心匿名、登录、跨用户、移动端和错误恢复 smoke QA。
- 零个未解决 P0/P1 缺陷；P2 缺陷必须有用户影响、规避方式、负责人和计划版本。
- 发布回滚经过演练，并能在 30 分钟内恢复到上一个已知可用版本；不可逆数据迁移不得与应用回滚不兼容。

## 11. 用户可用性门槛

- 至少 5 名不参与开发的中文目标用户完成可用性验收。
- 至少 80% 的参与者在无开发者代操作的情况下完成：登录、开始课程、完成并保存一次练习、找到历史、导入一个受支持素材、确认草稿和删除该素材。
- 所有参与者都能区分“草稿/练习反馈”与“正式转写/正式评分”。
- 阻断任务完成的问题必须修复并重新测试；结论、样本和修复证据写入发布记录。

## 12. 发布证据包

正式版 V1 必须保存以下可审计证据：

- 最终范围和状态矩阵；
- Git commit、PR、CI 和生产部署链接；
- 数据迁移、RLS、越权和 Secret 检查结果；
- 音高/节奏基准数据说明与结果；
- 浏览器、真实设备、可访问性和性能报告；
- 数据导出、删除、备份恢复和发布回滚演练记录；
- 音乐教育内容审核记录；
- 已知问题、风险接受和后续版本清单。

证据必须来自真实执行；静态源代码审查、mock、计划或未填写模板不能标记为 `PASS`。

## 13. 状态与变更控制

状态只允许：

- `NOT_STARTED`：尚未开始；
- `IN_PROGRESS`：已有工作但未达到全部门槛；
- `BLOCKED`：缺少外部权限、资源或决策；
- `PASS`：门槛全部满足且证据可访问；
- `DEFERRED`：经明确版本决策移到后续版本；
- `OUT_OF_SCOPE`：明确不属于 V1。

新增 V1 必须能力、降低质量阈值、把 `BLOCKED` 改成 `DEFERRED` 或接受 P0/P1 风险，必须记录原因并由产品所有者明确批准。普通实现细节由开发流程自主决策。

## 14. 最终退出条件

只有同时满足以下条件，才可以宣布“正式版 V1 完成”：

1. 所有 V1 `MUST` 项为 `PASS`，没有 `BLOCKED` 或未说明状态。
2. 条件能力已经通过门槛，或明确保持降级/非评分路径。
3. 所有 P0/P1 为零，P2 已记录并可接受。
4. 发布证据包完整，真实设备和用户验收已执行。
5. 最终代码合并到受保护主分支，适用的生产 Web 部署、签名 Android APK 和 post-deploy / post-install QA 通过。
6. 临时部署访问、测试账号、数据库高权限 Secret 和调试开关已恢复、删除或降权。
7. `docs/final-release-status-matrix.md` 的最终快照与生产事实一致。
