# P104 Android 算法与真机证据采集指南

本指南用于把第一代 Android 私测候选从“自动构建可用”推进到“真实设备与真实人声证据足够”。它不会上传录音，也不会因为表格填满就自动宣称算法专业准确；原始记录、授权与人工复核仍必须真实存在。

## 1. 当前门槛

- 合成或乐器单音至少 200 段；
- 真实人声至少 100 段，来自至少 20 位自愿参与者，覆盖至少 4 类采集设备；
- 覆盖常见声区、强弱音、颤音和背景噪声；记录授权、来源、采样率、标注版本和排除原因，调参集与最终验收集分开；
- 真实人声绝对音高误差：中位数不超过 35 音分、P95 不超过 100 音分、八度错误率不超过 2%；
- 低/中/高性能三台 Android 真机全部完成安装、飞行模式、麦克风、录音、P102 存储、P103 观察和后台释放；
- 每台飞行模式冷启动 P95 不超过 3 秒，麦克风到视觉反馈 P95 不超过 150 ms，并完成 20 轮稳定性；
- 至少 2 名具备视唱练耳或声乐教学经验的审核者批准观察文案与阈值适用边界。

## 2. 隐私边界

真实录音只保存在已获授权的本地加密位置；不得提交 Git、上传聊天、写入 APK、放入公开网盘或发送给未获授权者。样本 ID 使用随机编号，不记录姓名、电话、邮箱、学校、住址或精确位置。仓库只接收不含个人身份的汇总数字。

## 3. 仓库工具

1. 将 `local-fixtures/android-p104/evidence.example.json` 复制为同目录的 `evidence.local.json`；该文件已被 Git 忽略。
2. 只按真实结果填写候选版本、摘要、汇总指标和三台设备记录。
3. 运行 `npm run validate:android-p104-evidence`。
4. 输出中的 `BLOCKED` 是尚缺证据，不是软件崩溃。所有项目为 `PASS` 后仍需人工核对原始证据。

真实声音原始样本继续使用 `local-fixtures/real-voice/` 的本地约定和 `npm run validate:local-real-phone-comparison`，不得把单个 smoke 样本冒充统计基准。

## 4. 版本与签名

P104 真机活动开始前，应把功能冻结候选升级为 `0.3.0` / versionCode `3`，并切换到产品所有者控制的稳定私测签名。debug 签名切换到专用签名通常需要卸载，P102 本机曲线与录音会丢失；JSON 导出不含录音。密钥不得进入仓库、日志、截图或公开工件，至少保留两份加密离线备份。

P105 回滚使用同一签名且 versionCode 更高的“前向回滚包”，不能依赖安装更低 versionCode。最终版本号只在 P104 原始证据通过并完成签名升级演练后冻结。

## 5. 诚实结论

P104 只有在数据、三档真机与教育审核全部到位后才能标记完成。在此之前，P103 的周期性摆动、波动和尾音结果始终是探索性非评分观察；不能宣传为专业颤音分析、正式音域、声部鉴定或医学判断。
