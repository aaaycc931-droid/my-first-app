# P115e：七和弦密集 / 开放排列听辨验收标准

状态：**IMPLEMENTATION CANDIDATE / 本地候选**
日期：2026-07-20
QA level recommendation：**strict**

## 1. 目标

在 Android 离线练耳入口交付 P115 的下一个完整纵向切片：用户主动播放确定性的四声部七和弦，辨认密集排列或开放排列，再进入统一 Activity 非评分答案、本机复练和最小学习事实闭环。P115d 已通过 PR #383 合并到 `main@f0ea2b9`；该合并事实只作为 P115e 的开发基线，不证明本候选已经通过 CI、Android 真机或教育审核。

## 2. 题库与难度

- 基础：大七、属七，覆盖 12 个根音，只含原位，每题有密集 / 开放两种排列，共 48 个稳定变体。
- 进阶：大七、属七、小七、半减七，覆盖 12 个根音，只含原位，每题有密集 / 开放两种排列，共 96 个稳定变体。
- 挑战：上述四种性质，覆盖 12 个根音与原位、第一、第二、第三转位，每题有密集 / 开放两种排列，共 384 个稳定变体。
- 稳定目标使用 `seventh-chord-spacing:<root>:<quality>:<inversion>:<spacing>`；同一会话完整遍历前不得重复，相同种子、题号与稳定目标必须复现相同题目。
- 密集排列复用 P115d 的四声部七和弦。若密集音高为 `[bass, second, third, top]`，开放排列固定为 `[bass, third, second + 12, top + 12]`；由此保持同一音级多重集、和弦性质、转位与最低音，并保持实际频率严格递增。
- “密集 / 开放”只按低音上方三个声部判断：密集排列的上方相邻声部之间不留可容纳和弦音的空位，开放排列至少有一处该类空位。总跨度 `< 12` / `> 12` 半音仅作为当前固定映射的生成器回归性质；不得仅通过音色、时长、音量或播放顺序泄露答案。

三档题量均达到长期路线每档至少 40 个稳定题目或等价组合的数量门槛；这不替代两名视唱练耳教师对排列术语、听感、难度递进和解释的审核，也不表示完整 P115 已完成。

## 3. 真实使用路径

1. 用户选择基础、进阶或挑战，主动播放当前四声部七和弦。
2. 用户只选择“密集排列”或“开放排列”；七和弦性质与转位作为题目结构和解释，不作为本切片的作答项。
3. 查看答案后锁定选择，显示排列跨度、最低音保持与声部展开方式的中文非评分解释。
4. 错题以 `seventh-chord-spacing` 稳定目标写入本机复练；复练答对后移除。
5. 查看答案与开始复练只写入该题型的最小学习事件，不推断能力等级。
6. 新题、难度切换、复练目标变化、后台生命周期和组件卸载必须停止既有音源；返回后不得自动播放或提交。

## 4. 迁移、故障与隐私

- 复练队列升级到 schema/catalog v7，严格迁移 v1–v6；旧版本不得伪造 `seventh-chord-spacing` 目标。
- 学习画像信封升级到 schema v6，迁移 v1–v5，并为 `seventh-chord-spacing` 事实补零。
- 损坏、超限、重复目标、未知字段、未知版本、未知排列或与难度不匹配的稳定目标必须失败关闭。
- Web Audio 不可用、创建失败或播放失败时显示中文错误；storage 读写或移除失败必须提示，不能伪称已持久化，也不能阻断核心练习。
- 不保存音频、录音、用户具体选择、账号、设备标识、分数、能力标签或伙伴关系；不上传、不联网、不写数据库。

## 5. 自动验收

- P115e 排列生成器 focused tests：三档题量、稳定 ID、性质 / 转位 / 最低音保持、音级多重集保持、严格升序与跨度边界。
- Activity adapter、移动真实挂载行为与答案锁定回归。
- 扩展题库、复练队列 v1–v7、学习画像 v1–v6 与移动存储迁移回归。
- `npm run validate:android-local`
- `npm run check`

自动测试与本地构建不替代 PR Actions、Android System WebView、真实扬声器听感、后台恢复、长循环或教育审核证据；未实际执行的检查必须继续标记为未执行。

## 6. 不包含

- 七和弦性质或转位作答、调制、转调、和弦外音、完整四部和声写作或自由排列编辑器；
- 麦克风 / MIDI 作答、正式评分、自适应难度、跨设备同步或伙伴运行时；
- 两名教师审核、五名目标用户、三档 Android 真机矩阵、正式签名或最终发布完成。
