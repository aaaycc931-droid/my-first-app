# P117a — 旋律听写屏幕钢琴答案输入验收

最后更新：2026-07-21

状态：**当前开发边界 / implementation candidate**

QA level recommendation：**strict**

## 1. 目标与用户价值

P117a 在现有版本化三音旋律听写中增加屏幕钢琴真实答案输入。用户必须先完整听完当前隐藏旋律，再明确开始本轮钢琴作答，按听到的顺序逐个弹出三个音，主动检查后查看有序音高对照并复练。

本切片复用已有三难度题库、`activity-definition-v1` / `activity-session-v1`、`note-event-v1`、屏幕钢琴音频、本机复练和学习画像。它不是把现有参考钢琴改一个标签，也不是把任意钢琴弹奏静默收集为答案。

完整路径：

```text
选择旋律听写
→ 完整播放隐藏三音旋律
→ 明确开始屏幕钢琴作答
→ 逐个弹出三个音
→ 检查当前有序答案
→ 显示逐位置非评分对照
→ 重听复练或进入下一题
```

## 2. 当前切片范围

- 继续使用现有基础、进阶、挑战三档版本化三音旋律题目，不新增另一套题库。
- 继续保留现有音名 `choice` 与固定唱名 `solfege` 答案；屏幕钢琴使用真实 `piano` Activity answer。
- 只接收用户明确开始本轮作答后，由触摸、鼠标或键盘触发的屏幕琴键 `note-on`。
- 答案固定为三个有序音高；重复音必须通过再次起音保留其位置，不能去重。
- F♯4、C4 与 C5 必须从题目、屏幕琴键事件、Activity answer、检查和中文解释中无损往返。
- 第三个有效起音进入答案后自动停止接收；用户主动检查前可以清空，但清空必须同时作废旧播放资格与 Activity attempt，重新完整播放后才能开始新答案，不能把旧资格转移到新 attempt。
- 检查只返回 `consistent`、`different` 或 `insufficient`，并始终携带 `assessmentMode: "non-scoring"`。
- 错题继续只向现有本机复练队列写入稳定题目标识；学习画像只记录已检查的一致／不同事实，不保存琴键序列、声音、NoteEvent、ActivitySession 或检查详情。

P117a 优先挂载在 Android 扩展本地随机练习入口。固定复练题与定制题保持现有音名／固定唱名流程，避免为答案方式新增持久化字段，也避免已知目标跨活动复用。Web `/practice` 的共享旧入口不因本切片自动宣称已完成屏幕钢琴输入。

## 3. Activity 与屏幕钢琴输入协议

活动定义必须满足：

- `family: "melody-dictation"`；
- `allowedInputModes` 包含 `piano`；
- canonical target 继续使用当前题目的三个稳定 note id 和 ordered 策略；
- `activityId`、`targetId`、content version、难度和题目 `variantId` 可追溯且不依赖随机时间戳；
- `source.kind: "built-in"`、`source.reviewState: "confirmed"`、`assessmentMode: "non-scoring"`；
- `piano` answer 使用 `{ mode: "piano", noteIds }`，并真实驱动 ready、answering、checked 与 restart 生命周期。

输入适配必须复用现有共享音乐事件边界：

- 只记录 producer 已确认为屏幕钢琴用户输入的 `note-on`；
- `note-off`、sustain、all-notes-off、题目播放、参考播放、演奏回放、压力测试和自动谱面播放都不能写入答案；
- Web MIDI、Android USB MIDI 与 BLE MIDI 不得冒充 `piano` answer；
- 音高按稳定 sounding note id 归一化，不能由界面标签、设备名称或当前视觉位置猜测；
- 当前答案音域必须覆盖题目实际使用的 C4–C5 与 F♯4；移调不为零时不得开始作答，不能把移调后的琴键身份静默写入目标音高；
- 同一琴键连续重复音必须由两个独立 `note-on` 表达；并发输入仍按规范化事件顺序处理，达到三个音后拒绝迟到起音。

现有 P114d 原创谱面跟弹面板会显示可见目标，不能原样挂入隐藏旋律听写。P117a 可以复用其事件适配器与 session 语义，但必须提供不提前显示目标的专用作答表面。

## 4. 完整播放资格与 fail-closed

屏幕钢琴作答只有在当前题目音频时间线完整播放结束后才可解锁。仅调用播放函数、创建 oscillator、经过预计 wall-clock 时间或播放了一部分，均不能授予资格。

实现必须满足：

- 播放 token 同时绑定当前题目、当前 attempt 和当前完整时间线；
- 播放完成前，开始作答、琴键答案捕获与检查保持禁用，并显示简体中文原因；
- AudioContext 必须在预期结束点仍为 `running`，且 `currentTime` 已到达完整旋律终点；
- 重播、手动停止、全局停止、后台、失去音频上下文、切换答案方式、切换难度、换题、重置与卸载都会撤销资格并清除旧答案和 checked 证据；
- 迟到完成计时器、迟到音频准备、迟到 NoteEvent 或旧 revision 不能重新解锁或改写新 attempt；
- 播放准备或本地声音失败时显示中文恢复提示，不生成空答案或假完成；
- 进入钢琴作答前题目声音必须已经结束；重播题目时必须停止所有钢琴声部并作废当前答案。

屏幕琴键为了发出自己的声音而执行的受控音频切换不能被误判为一次外部中断并立即清除合法答案资格；用户触发的全局停止、页面离开和后台生命周期仍必须 fail closed。实现需要用明确的运行阶段或来源区分两者，不能靠忽略所有 stop 事件规避清理。

## 5. 用户状态与交互

| 状态 | 必须行为 |
| --- | --- |
| 题目准备中 | 播放、钢琴作答和检查禁用，显示准备状态 |
| 等待完整播放 | 只允许播放题目；说明完整听题后才能弹奏答案 |
| 正在播放 | 目标保持隐藏；允许“停止并作废”，不接收答案琴键 |
| 播放已完成 | 显示“开始屏幕钢琴作答”，仍不提前显示目标 |
| 正在接收 | 显示当前已输入 `0–3` 个位置；只接收合格屏幕 `note-on` |
| 三音已完成 | 自动停止接收，允许主动检查；清空会作废资格并要求重新完整播放 |
| 部分答案 | 检查返回 insufficient 或保持禁用，并说明还差几个音 |
| 已检查 | 锁定本轮输入，显示目标与用户答案的有序对照、中文解释和非评分边界 |
| 已作废 | 清除资格、答案和旧证据，说明必须重新完整播放 |

至少提供：播放题目、停止并作废、开始作答、检查本轮、清空并重新听题、重新播放并复练、下一题。所有 disabled 操作必须有可理解原因。

结果说明必须保留音高顺序与重复音，并明确“与本题答案一致／不同”。不得把一致写成“通过”“正确率 100%”或能力结论；不同也不能写成考试失败、等级下降或演奏能力不足。

## 6. 兼容、隐私与存储边界

- 现有音名与固定唱名 ordered 答案、三难度题目、调度器、参考钢琴和本机复练兼容行为保持通过。
- 答案方式、题目、难度或 attempt 变化后，旧 answer 与 check evidence 必须失效。
- 当前琴键输入、NoteEvent、声音与 ActivitySession 只在内存中；不写入 localStorage、IndexedDB、账号、数据库或云端。
- 本机复练只保存复现原题所需的既有题型、难度、种子、序号和 `variantId`；不迁移或扩展现有存储 schema。
- 学习画像继续只表达最小已检查事实和复练建议原因，可关闭建议并独立重置；不得把屏幕钢琴结果转成正式技能等级。
- 不新增账户、上传、云端、数据库、私有音频存储或依赖。

## 7. 明确非目标与后续边界

P117a 不包含：

- 五线谱答案输入；它保留给 P117b；
- 简谱答案输入；它保留给 P117c；
- Web MIDI、Android USB MIDI、BLE MIDI 或外接键盘硬件答案；
- 旋律回唱、旋律视唱、麦克风、录音、实时曲线、P113 提交后分析、逐句反馈或分段复练；
- 预备拍、节拍同步、节奏时值听写、四音以上旋律或新课程题库；
- 完整制谱、MusicXML、OMR、导入导出或正式谱面；
- 分数、百分比、等级、排名、通过／失败、正式考试或教育能力结论；
- 伙伴 UI、生成式答案、账号、上传、云端处理、数据库迁移或新的持久化格式。

P117b / P117c 完成五线谱／简谱输入也不自动证明 P117 整体完成。旋律回唱、旋律视唱、预备拍、实时与提交后分析仍需后续独立切片和真实人声证据。

## 8. 自动测试与门禁

P117a 候选至少需要：

- focused adapter tests：`piano` 定义、三音 ordered、重复音、F♯4/C5、一致／不同／证据不足、非法来源、输入上限、checked 锁定、revision 与 restart；
- focused playback qualification tests：未完整播放拒绝、完整时间线授予、AudioContext 中断、重播／停止／全停／后台／卸载作废、迟到计时器与旧 token 拒绝；
- 真实挂载移动端行为测试：完整播放前锁定、开始接收、逐音计数、三音自动停止、主动检查、结果揭示、重置、复练、换题、难度／答案方式切换；
- 输入污染回归：参考播放、谱面回放、压力测试、note-off、踏板、Web MIDI、USB/BLE 事件均不能进入屏幕答案；
- 现有音名、固定唱名、三难度题库、复练队列、学习画像、钢琴音频、全局停止与 Android lifecycle 回归；
- `npm run check`、Android sync、Android validator、GitHub `quality` 与 `android-local`；
- Debug APK 构建、独立工件校验和 SHA-256 记录。

自动测试与 CI 只证明代码、构建和约定行为，不证明真实 Android 触控、扬声器、音频焦点、后台恢复、长时间稳定性、教师审核或目标用户可用性。

## 9. 手动与外部证据

QA level recommendation：**strict**。实现合并前后仍需分别记录：

- Android 真机逐音触控、重复音、快速输入、多指误触、扬声器与钢琴发声；
- 真机重播、全停、返回键、前后台、音频焦点中断和进程重建；
- 至少三档目标 Android 设备上的离线冷启动和长循环；
- 双教师对题目真值、答案顺序、难度递进和解释文案的审核；
- 目标用户对“先完整听题—弹奏答案—检查—复练”流程的可理解性与误触恢复。

在相应证据实际执行并入库前，上述项目必须保持未完成。源码审查、DOM 测试、模拟音频、CI 或 APK 工件不能替代浏览器手动 QA、Android 真机、教师和目标用户证据。
