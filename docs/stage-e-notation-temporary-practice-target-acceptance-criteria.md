# Stage E — 手动乐谱草稿临时练习目标验收标准

## 1. 阶段目标

Stage E 只接收当前会话内、已检查且已通过 Stage D 小节时值校验的 Stage B 手动乐谱片段草稿。用户必须查看确认摘要并主动确认，系统才可生成一个 browser-local、session-only、non-scoring 的临时视唱或节奏练习目标。

## 2. 创建条件与流程

- 草稿必须非空、非 stale 且已检查。
- 当前 Stage D 结果必须为 valid，并且仍对应当前草稿指纹。
- 用户可选择“临时视唱练习目标”或“临时节奏练习目标”。
- 创建前必须显示来源、拍号、两小节范围、事件数量、检查 / 校验状态和目标类型。
- 用户点击“确认生成”才创建；不自动进入练习。

## 3. 临时练习行为

- 临时目标保存事件类型、音高（如有）、时值、小节、拍号、来源说明与创建时刻。
- “临时乐谱练习”功能区按事件顺序显示当前事件，并支持上一个、下一个和从头开始。
- 视唱目标提示按音高视唱；节奏目标提示按时值拍读或拍击；休止符明确提示保持休止。
- 当前切片不自动播放、不录音、不调用麦克风、不生成评分、等级、通过 / 失败或正式成绩，也不替换既有“本地旋律”流程。

## 4. 失效、清除与会话边界

- 修改、删除、清空、重置草稿；改变检查状态；来源图片变化；校验结果变为 stale / invalid / cleared，均会使临时目标 stale。
- stale 目标不能进入临时练习；用户可以清除旧目标，重新检查、校验并确认创建。
- 手动清除目标不清除原草稿或 Stage D 结果。
- 草稿、校验、目标和练习位置都只在当前页面内存；刷新后消失，不写入 localStorage、IndexedDB、数据库或云端。

## 5. 文案与禁止事项

- 用户可见文案默认使用简体中文。
- 必须说明这是临时目标，不是正式识谱、正式转写、最终目标或评分依据。
- 不实现 OCR / OMR、图片像素分析、PDF 解析、Audiveris、上传、`/api/recognize`、账号、数据库、云端或新依赖。

## 6. 最低自动测试

- 未检查、无校验、无效 / stale 校验不能创建。
- 有效草稿可创建两种目标类型，并保留事件快照。
- 草稿修改、校验清除或来源失效会让已有目标 stale。
- stale 目标不能继续进入练习。

## 7. QA 建议

QA level: strict。需要 focused tests、Stage B / C / D 回归、生产 build、远端 PR checks，以及合并后在公开生产站点完成浏览器交互 QA。
