# P116b 节奏回模验收

最后更新：2026-07-20

P116b 是 P116 节奏专业套件的第二个最小完整纵向切片。它复用版本化节奏题库、Activity `tap` 输入、单调动态规划逐拍反馈、本机复练与学习画像，提供“听隐藏目标—完整播放后解锁—预备拍—触控回模—揭示逐拍非评分反馈”的离线闭环。

## 用户闭环

1. 用户从 Android 扩展节奏听辨入口进入节奏回模；目标在听题和点击阶段保持隐藏。
2. 用户完整听完一小节本地合成节奏后，“开始节奏回模”才可用；未听完时不能提交点击。
3. 用户跟随一小节预备拍，在四拍窗口内使用屏幕触控复现节奏。
4. 本轮结束后才显示目标起点，以及接近、偏早、偏晚、漏掉或额外点击。
5. 有足够证据时只把最小一致／不同事实送入现有复练队列与学习画像；不保存原始触控序列。

## Fail-closed 边界

- 每次听题与回模均绑定递增 runtime token；迟到播放完成与计时器不能恢复旧状态。
- 重听、手动停止、清除、页面切换、后台、全局音频停止与卸载都会清除目标、点击、计时器和旧反馈。
- 重听会使已检查 attempt 失效并建立新 attempt；旧 attempt 不得写入复练或画像。
- 本地声音无法准备时回到未听题状态，并显示中文可恢复原因。

## 产品与证据边界

- 仅使用 Web Audio 本地合成、屏幕触控与当前页面内存；不读取麦克风，不上传音频或点击，不新增账户、云端、数据库或依赖。
- 反馈明确为非评分，不显示总分、百分比、等级、通过／失败或演奏能力判断。
- 自动测试只验证确定性状态机与 React 挂载行为，不证明真实 Android 设备触控、真实拍手、教师审核或教育有效性。
- P116a 的八样本会话校准仍是相邻的独立练习工具；P116b 不静默继承或持久化另一 attempt 的校准值。

## 自动验收

- focused domain tests：目标时间映射、三档版本化题目、`rhythm-imitation` Activity、接近／不同／证据不足和无评分字段。
- 真实挂载移动 React 行为：完整听题前禁用回模、点击窗口、结果后揭示、全局停止与迟到计时器 fail closed。
- Android App 行为回归：现有扩展节奏入口同时真实挂载 P116a 与 P116b，旧 choice Activity、复练和画像保持兼容。
- 本切片不改变本地存储 schema，因此没有数据迁移；旧复练目标仍由现有迁移测试覆盖。
- Android validator：检查本地源码、Activity、隐藏目标、runtime token、全局停止、隐私与非评分文案。
- 合并前运行完整 `check`、Android sync／validator；GitHub `android-local` 是 API 36 Debug APK 的权威自动门禁。

QA level recommendation：strict。该切片改变本地音频播放、计时、重播／停止、练习目标与反馈状态。
