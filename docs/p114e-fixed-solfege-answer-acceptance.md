# P114e — 三音旋律听写固定唱名答案接入验收标准

状态：**PR #371 已 squash merge / main `b8cff79626af3267611291b13f020a24f5a55ff5`**

QA level recommendation：**strict**

## 1. 目标与用户价值

P114e 在现有三音旋律听写真实练习上增加“固定唱名”作答方式，并把它作为 `activity-definition-v1` / `activity-session-v1` 的首个真实 `solfege` 输入 vertical slice。用户仍先播放同一组三音短旋律，再选择使用现有音名答案或固定唱名答案，按听到的顺序填写三个位置，查看非评分解释并复练。

本切片必须保留原有音名 `choice` 路径；固定唱名不是新题库、不是独立课程，也不能通过复制一套题目状态造成两条行为不一致的练习流程。

## 2. 输入模式与稳定映射

活动必须明确区分当前答案模式：

- 音名模式继续提交 `{ mode: "choice", optionIds }`；
- 固定唱名模式提交 `{ mode: "solfege", tokens }`；
- 两种模式都按三个位置的先后顺序检查，不能使用无序集合；
- 重复音必须保留每次出现的位置，例如 `do4 → re4 → do4` 不能压缩或去重；
- 活动可以继续以现有有序 `choice` 音名序列作为唯一 canonical target，同时把 `solfege` 加入允许输入；固定唱名检查 adapter 必须先按冻结映射还原为对应音高再逐位置比较，不能把唱名 token 塞入 `choice.optionIds`，也不能直接拿 token 与音名 id 比较。

固定唱名 token 必须稳定、可序列化、带足够的音高身份，至少覆盖当前扩展三音旋律题库：

| 音高 | 用户可见固定唱名 | 建议稳定 token |
| --- | --- | --- |
| C4 | do | `do4` |
| D4 | re | `re4` |
| E4 | mi | `mi4` |
| F4 | fa | `fa4` |
| F♯4 | 升 fa | `fi4` |
| G4 | sol | `sol4` |
| A4 | la | `la4` |
| B4 | si | `ti4` |
| C5 | 高音 do | `do5` |

实现必须通过 focused tests 冻结该 token 映射。`fi4` 是 F♯4 的内部固定唱名 token，`ti4` 是 B4 的内部 token；用户界面仍按本项目中文术语显示“升 fa”与“si”。特别是 F♯4 不能退化成 F4，C5 不能与 C4 共用 `do4`；界面显示可以简洁，提交值必须保留升号与八度身份。

切换音名／固定唱名时必须停止当前播放，清空当前三个位置、答案显示和旧检查证据，并开始边界清楚的新尝试。不得把一种模式下已经填写或检查的答案静默转换为另一种模式的已完成答案。

## 3. 完整用户流程

### 3.1 准备与空状态

- Web `/practice` 与 Android 共享入口继续进入现有三音旋律听写，不增加空入口。
- 题目尚未准备好时，播放、作答、检查和下一题按现有调度规则禁用，并显示简体中文准备状态。
- 三个答案位置都为空时，必须提示用户依次填写三个音；不能生成检查结果。
- 只填写一个或两个位置时，“查看本题答案”保持禁用，并说明还需完成全部位置。

### 3.2 作答与锁定

- 三个位置分别显示“第 1 个音”“第 2 个音”“第 3 个音”。
- 每次选择只修改当前位置；允许相同 token 出现在多个位置。
- 答案揭示后，本轮输入锁定，不能继续改变旧答案或产生第二份相互矛盾的检查证据。
- 检查结果只允许“与本题答案一致”“与本题答案不同”或“当前证据不足以可靠检查”等非评分表达。
- 答案说明必须显示正确的有序序列和用户提交的有序序列；固定唱名模式显示固定唱名，不应在用户答案区域退回为内部 token。

### 3.3 复练、重置、下一题与难度

- 答案不同后，“重新播放并复练本题”开始新尝试、清空旧选择与检查证据，并重新播放同一版本化题目。
- “重置本题”停止播放，清空本轮选择、答案、错误提示和协议检查状态，但保留当前题目与当前答案模式。
- “下一题”停止播放并清空本轮状态；随机练习进入调度器的下一题，复习队列入口按现有行为返回随机练习。
- 基础、进阶、挑战难度继续使用现有版本化三音题库和调度规则；切换难度必须清空旧答案和检查证据并从该难度的新序列开始。
- 挑战难度中的 F♯4 与 C5 必须能在固定唱名选项、提交答案、正确答案和解释中完整往返。

### 3.4 本机复习队列

- 音名与固定唱名两种模式的答题结果都继续复用现有最小本机复习队列语义。
- 答错时只保存复现原题所需的现有最小题目标识，不保存声音、自由文本或整份活动会话。
- 从复习队列进入时，题目 `variantId`、难度和种子必须继续稳定复现；难度选择保持锁定。
- 离开复习目标、重试、重置和下一题不能制造重复队列写入。
- P114e 不迁移、不扩展或破坏本机复练 v2 存储格式；若答案模式无需持久化，就不得为此改动旧格式。

## 4. 协议与兼容性要求

- 现有 `choice` 活动定义、稳定题目身份、Web Audio 播放、课程 RPC 和本机复练 v2 行为保持兼容。
- 固定唱名路径必须真实驱动 `ActivitySessionV1` 的 ready、answering、checked 和 restart 生命周期，不得只在 UI 中换标签。
- ordered 检查必须同时覆盖音名与固定唱名；`do4 → re4 → mi4` 与 `mi4 → re4 → do4` 必须判为有差异。
- 题目中的重复音、F♯4 和 C5 必须从题库到活动定义、提交、检查和解释完整保留。
- 旧的 checked 结果在答案模式、难度、题目或尝试变化后必须失效。
- 所有用户可见文案使用简体中文；`do`、`re`、`mi`、`fa`、`sol`、`la`、`si` 属必要音乐术语，内部 schema/token 名不得直接暴露给用户。

## 5. Web 与 Android 验收

同一共享练习组件必须在 Web `/practice` 与 Capacitor Android 本地静态包中可用。至少验证：

1. 两端均可选择音名或固定唱名并完成三位置作答；
2. 播放中切换模式、难度、重置、复练或下一题不会留下声音；
3. Android 返回前台、组件卸载和全局停止音频后不出现残音或旧答案复活；
4. 小屏下三个位置、模式选择、检查、复练、重置和下一题可操作且文案不被截断到不可理解；
5. 无网络时 Android 核心题目、Web Audio 和答案检查不依赖生产网站、Supabase 或云端接口。

桌面浏览器自动测试或源码审查不能替代 Android 真机音频、触控和生命周期验证。

## 6. 自动测试与质量门禁

P114e 实现候选至少需要：

- focused 纯映射测试：九个当前音高与固定唱名 token 一一对应，重点覆盖 F♯4、C5 和重复音；
- focused adapter/session 测试：`choice` 与 `solfege` 定义、ordered 检查、一致／有差异／证据不足、新尝试和 revision 防护；
- 三难度题库回归：每档版本化组合可生成，挑战题固定唱名不丢升号或八度；
- 共享真实挂载行为测试：empty、部分填写、完整填写、检查锁定、模式切换、难度切换、复练、重置、下一题和复习队列；
- 现有单音、音程、节奏、三音旋律 `choice` 协议回归；
- `npm run check`；
- Android 本地静态边界验证和 CI focused 门禁。

QA level recommendation：**strict**。自动门禁通过后仍需分别记录 Web 浏览器手动 QA 与 Android 真机 QA；没有执行时必须明确写“未执行”，不得用自动测试冒充。

## 7. 非目标

P114e 不包含：

- 五线谱或简谱答案编辑器；
- movable-do、首调唱名、调式分析或唱名教学课程；
- 麦克风、屏幕钢琴、USB MIDI 或 BLE MIDI 新答案适配；
- 新题库、四音以上旋律、和弦、节奏时值输入或正式视唱评分；
- 总分、百分比、等级、排名、通过／失败或正式考试结论；
- 伙伴 UI、自由聊天或生成式唱名；
- 上传、云端分析、账号 schema、数据库迁移或新的持久化格式；
- Web Audio 音色重构、钢琴运行时改造、MusicXML/OMR、制谱或伴奏；
- 新依赖或 package 变更，除非后续出现独立、明确且经批准的技术决策。
