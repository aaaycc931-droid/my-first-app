> 规范关系：本文件保留长期功能域摘要；2026-07-18 后的 canonical 八产品终局见 `docs/eight-product-unified-competitive-roadmap-2026-07-18.md`，P113 之后包含伙伴系统的统一执行波次见 `docs/unified-development-roadmap-with-ai-music-companion-2026-07-18.md`。正式版 V1 的唯一范围、量化门槛和退出条件仍见 `docs/final-release-definition-of-done.md`，实时完成状态见 `docs/final-release-status-matrix.md`。

## 产品承诺

本产品面向中文学习者，终局是一套同时覆盖练声/音准、视唱练耳、乐理、专业钢琴、制谱/识谱、伴奏/歌曲工作台、学习系统与 AI 音乐伙伴的全能音乐学习和创作产品。伙伴依据可信音乐证据解释和连接工具，不替代底层测量；关闭伙伴后所有核心功能仍完整可用。当前优先交付无需访问生产网站的 Android 本地 APK，同时保留并发展 Web 私有学习平台与中国区合规云端；离线优先是阶段约束，不是最终卖点。产品不提供社区、公开个人页、公开上传、公开曲库、关注、评论或私信。

正式版本的主路径是：

```text
选择课程或私有素材
→ 预览并检查自动草稿
→ 确认练习目标
→ 听、唱、拍或弹
→ 查看可解释反馈
→ 复练薄弱点
→ 保存进度与下一步建议
```

## 正式功能域

| 功能域 | 用户获得的能力 | 正式完成标准 |
| --- | --- | --- |
| 账户与隐私 | 登录、设备间同步、数据导出与删除 | 每条私有数据都有所有者；删除链路覆盖数据库、对象存储与任务产物 |
| 课程与练耳 | 单音、音程、和弦、节奏、旋律听写、难度与复练 | 题库、答案、解释、错题复练、课程进度均可持久化 |
| 视唱与节奏 | 目标播放、节拍、录音、实时辅助、回放、分句复练 | 每次尝试可关联目标、输入、版本化评测和反馈 |
| 钢琴与 MIDI | 专业屏幕钢琴、多音色、节拍器、录音、MIDI 与谱面学习 | 移动端低延迟、真机多指/设备恢复和学习闭环有证据 |
| 乐谱、制谱与 OMR | 创建/编辑/播放，MusicXML/MXL/MIDI/PDF/图片导入导出，图片/PDF/音频草稿修正 | round-trip、撤销恢复和原谱对照有证据；自动结果先预览检查确认 |
| 伴奏与歌曲工作台 | 私有歌曲导入、分离、多轨编辑、变调变速、分析、跟唱跟弹和导出 | 质量/同步/伪影/失败恢复有基准；来源、许可和数据所有权清晰 |
| 评测与反馈 | 音高、节奏、起音、稳定性、分句与综合反馈 | 有版本、置信度、适用条件和证据；低置信度不产生强结论 |
| 学习系统 | 记录、技能画像、复习队列、目标与趋势 | 推荐可解释，可关闭，不以黑箱分数替代反馈 |
| AI 音乐伙伴 | 依据真实证据解释、范唱、发起复练并受控调用钢琴/乐谱/伴奏等工具；单伙伴先行，未来多伙伴 | 音乐事实与人格表达分层；工具有 allowlist、确认和撤销；可隐藏、可删除、低置信拒答；通过教学、安全、隐私、性能和用户验收 |
| 私有云处理 | 私有上传、异步处理、任务状态、结果与删除 | 仅账户本人可访问；有配额、保留期、失败恢复和审计 |

## 核心边界

- 用户可见文案默认使用简体中文。
- 自动识别、自动生成和自动分析的结果必须经过“预览 → 检查 → 修改 / 确认 → 再进入练习”。
- 正式评测先使用可解释的确定性评测内核；AI 只解释已结构化的结果，不替代测量。
- 伙伴的先天人格、学习风格和关系成长不得改写证据、成绩或核心学习权限；伙伴可完全隐藏或关闭。
- 云端音频与乐谱仅用于账户所有者的私有练习，不提供搜索、公开播放、下载或分享。
- 评分、等级、通过/失败等高影响结论必须建立在真实数据集、校准、版本化规则和人工音乐教育验证之上；在此之前只能显示诊断或练习反馈。
- 所有外部服务均通过服务端环境变量接入，前端不保存密钥。

## 推荐正式技术架构

| 层 | 选择 | 责任 |
| --- | --- | --- |
| Android | Vite 静态入口 + Capacitor 本地 WebView | 本地 bundle、签名 APK、离线核心练习、权限/生命周期与持续真机回归 |
| Web | Next.js App Router + TypeScript | 中文 Web 与移动端响应式界面 |
| 认证与数据库 | Supabase Auth + Postgres（行级安全） | 用户身份、私有记录、课程与权限 |
| 文件存储 | Supabase Storage 私有桶 | 乐谱、音频、导出文件与处理产物 |
| 后台任务 | 受认证的任务 API + 队列 worker | 转码、OMR、音频分析、报告生成 |
| 音频处理 | 浏览器实时辅助 + 云端离线评测 worker | 本地低延迟体验与可复核深度处理 |
| 识谱 | MusicXML 优先；隔离的 worker OMR | 可审阅乐谱草稿，禁止在请求路径直接跑重型 OMR |
| 观测与安全 | 审计事件、错误追踪、速率限制、内容/配额策略 | 可运营性与私有数据保护 |

具体供应商可替换，但数据库必须支持行级所有权控制、私有对象存储必须支持短时签名访问、后台任务必须可重试和可取消。

## 交付分层

1. **产品核心**：账户、私有数据、系统课程、持久化练习记录、钢琴和基础练习闭环。
2. **可信评测**：录音、评测版本、置信度、可解释反馈、回放与复练。
3. **私有素材**：MusicXML/MXL、图片/PDF、音频上传，异步处理、草稿审核和删除。
4. **学习系统**：技能画像、复习队列、目标、趋势与导出。
5. **正式发布**：安全审查、数据删除/导出演练、真实设备与真实录音评测、可访问性和灾难恢复。

本文件取代“最终产品形态未定义”的状态；它不意味着现有 P68 浏览器本地功能已具备上述正式能力。
