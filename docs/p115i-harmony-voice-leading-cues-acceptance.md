# P115i 和声低音／高声部线索与可解释反馈验收

状态：**本地严格验证中的 implementation candidate**  
基线：`main@4078b8e5d9fe0d3253b3b0f4feba16ede28eae2d`  
QA level recommendation：**strict**

## 最小完整纵向切片

1. 复用 P115b 已有和声进行题库、稳定目标、Activity、复练和学习画像，不新增题型或存储 schema。
2. 每题从实际三和弦排列确定最低音与最高音序列；用户可分别只听低音线索或高声部线索。
3. 核对答案后解释两条声部在相邻和弦间的“上行／下行／保持”，明确这些线索只解释听感，不单独判分。
4. 原和声进行完整播放和答案语义不变；线索播放不改变、提交或暗示答案。

## 边界

- 全部频率由既有本地题目确定，使用本地 Web Audio；不上传、不联网、不新增账户、数据库或录音。
- 不生成分数、等级、通过／失败、能力标签或正式和声学诊断。
- 本机复练与画像继续只保存既有和声进行稳定目标和核对事实，不保存播放过哪条线索。

## 验收门槛

- domain tests 校验低音/高声部序列严格来自每个和弦最低/最高频率，方向说明非空且带非评分边界。
- 真实挂载移动行为测试确认两条线索控件存在，核对后展示当前题的实际方向解释，复练与画像行为不回归。
- `npm run check`、Android sync、validator 通过；GitHub `quality` 与 `android-local` 通过并上传独立验证的 Debug APK 工件。

## 证据限制

自动测试、合成音、CI 与 APK 构建不等于教师对和声术语/难度的审核，也不等于 Android 真机听感、目标用户理解或教育有效性证据。双教师校准和真实设备测试仍是后续独立硬门槛。
