# Android 离线专业视唱练耳产品路线

状态：**Approved product direction / 第一代 Android 私测版执行路线**  
生效日期：2026-07-16  
适用范围：面向中文用户、可私下分发、断网可用的 Android APK

## 1. 产品结论

第一代产品不是题库展示器，也不是单纯调音器。它必须形成一个真实可用的日常练习闭环：

> 听目标 → 唱出来 → 实时看音高曲线 → 找到偏差 → 分段复练 → 保存或清除 → 再次练习

核心定位为：

- 实时音高可视化；
- 目标旋律对照与分段复练；
- 中文视唱、听辨和练耳题库；
- 本地钢琴和参考音；
- 无账号也可用、默认不上传声音的离线 Android 练习工具。

竞品功能只用于理解用户问题，不复制其界面、图标、文案或视觉资产。产品必须使用自有交互与信息架构。

## 2. 第一代 Android 私测版范围

### 2.1 必须完成

1. 本地实时麦克风音高检测与滚动曲线。
2. 自由练唱和目标跟唱两种模式。
3. 录音、停止、回放、清空和用户主动保存。
4. 单音、音程、音阶、练声音型和短旋律模唱。
5. 目标音块与用户音高曲线叠加。
6. “偏高 / 偏低 / 接近 / 无可靠人声”非评分反馈。
7. 局部选择、循环和分段复练。
8. 单音、音程、节奏和旋律听写三个难度层级。
9. 稳定题目标识、旧队列迁移和 Android 本机错题复练。
10. 独立及练习内嵌的本地参考钢琴。
11. 本地练习记录、备注、回看、单条删除和全部清除。
12. 飞行模式冷启动，不依赖 Web、Vercel、Supabase 或远程 `server.url`。
13. 简体中文主流程、完整错误恢复和 Android 真机验收。

### 2.2 第一代不做

- 应用商店、AAB 和上架材料；
- 支付、会员、社区和公开分享；
- 账号强依赖、云端同步和公开曲库；
- 图片 / PDF OMR 的正式 Android 用户入口；
- 整曲云端声音分离或私人歌曲云管线；
- 未通过数据基准和教育审核的正式分数、等级和通过 / 失败。

## 4. 实时音高系统架构

```text
Android 麦克风
→ 实时音频采集
→ 音高检测引擎
→ 置信度 / 无声判断 / 平滑 / 八度跳变抑制
→ 标准音高帧
→ 曲线渲染、目标对照、反馈和本地记录
```

标准音高帧至少包含：

- 单调时间戳；
- 频率 Hz；
- 浮点 MIDI 音高；
- 最近音名和对应 cents 偏差；
- RMS / 相对音量；
- voiced / unvoiced 标记；
- 置信度；
- 引擎与参数版本。

引擎必须通过可替换接口隔离。首先比较当前 autocorrelation 基线、YIN 和 McLeod Pitch Method。如果 Android System WebView 在中低端设备上无法达到延迟与稳定性门槛，改用 Kotlin `AudioRecord` 采集 / 检测，只向 React 界面传送紧凑音高帧。

## 5. 第一代练习模式

### 5.1 自由音准曲线

- 实时当前音名、Hz、cents 和置信状态；
- 最近 8–12 秒滚动曲线；
- 无可靠人声时不连线，显示“等待稳定人声”；
- 可用于长音、音阶、滑音和自由练声。

### 5.2 单音模唱

- 播放目标音 → 可选倒计时 → 用户演唱 → 实时对照 → 整段回看 → 重练；
- 目标音域使用用户可选范围，不用性别标签限制音域。

### 5.3 音程模唱

- 上行、下行和自由方向；
- 小二度至八度；
- 固定 / 随机根音；
- 全部参考或只播根音；
- 两个目标音可分别复练。

### 5.4 音阶与练声音型

第一批至少包含：

- 五声音阶；
- 大调音阶；
- `1-2-3-4-5-4-3-2-1`；
- `1-3-5-3-1`；
- 八度滑音；
- 半音移调循环。

所有音型使用版本化 manifest，并显式定义根音、方向、BPM、八度、循环和参考音。

### 5.5 旋律跟唱

- 三至八音短旋律；
- 目标音使用时间对齐的横向音块；
- 用户曲线与目标叠加；
- 完整练习、手动分段和循环复练；
- 点击问题片段即可重练；
- 只提供可解释诊断，不生成正式分数。

## 6. 界面原则

1. 顶部显示当前音名、cents 偏差和检测可靠性。
2. 中部是目标音块、用户曲线、音名网格和当前游标。
3. 底部提供开始、停止、回放、清空和循环。
4. 设置抽屉提供音域、A4 基准、最低收音、平滑级别、波形显示和音名系统。
5. 目标音、接近、偏高、偏低和不可靠状态使用稳定且可辨识的不同视觉通道，不只依赖颜色。
6. 不使用持续闪烁的红色错误诱导用户追逐瞬时抖动。
7. 保留 360 CSS px 小屏可操作性、可见焦点和屏幕阅读器名称。

## 7. 本地记录与隐私

录音结束后默认只保留在当前练习中。用户必须主动选择：

- 丢弃；
- 只保存曲线；
- 保存曲线和录音；
- 添加备注；
- 导出；
- 删除单条或清空全部。

永久记录至少包含练习类型与设置、时间、时长、目标版本、音高帧、算法版本、录音保存标记和用户备注。数据使用应用私有存储，不申请宽泛文件权限，不在后台默认上传。

## 8. 不得误导的专业边界

- 第一代反馈是练习诊断，不是正式分数、等级或通过 / 失败。
- 低置信度、无声、强噪声、复音和越界输入必须降级为“不足以判断”。
- 音高和音量曲线可用于观察稳定度、连续性和颤音趋势，不能直接诊断气息、声带状态或医学问题。
- “换声点”只能作为用户自主观察草稿，未经专业教育 / 声乐审核不给出强结论。
- 参考音播放时必须防止手机扬声器被麦克风当作用户人声；必要时暂停检测或引导使用耳机。

## 9. 算法与性能门槛

- 音高视觉反馈更新率为 10–20 FPS；
- 本地麦克风到视觉反馈延迟的目标 P95 不高于 150 ms；
- 清洁单音样本音高绝对误差中位数不高于 25 cents，P95 不高于 60 cents；
- 真实人声样本中位数不高于 35 cents，P95 不高于 100 cents；
- 真实人声八度错误率不高于 2%；
- 连续 20 轮录音、停止、回放、清空、切页和后台恢复后，不得残留媒体轨、计时器或可复现内存增长。

任何性能数字必须记录手机、Android、System WebView、采样率、样本量和日期，不得把 mock 或桌面浏览器结果写成真机结论。

## 11. 竞争力原则

1. 看见问题后可以直接进入对应复练，而不只是展示曲线。
2. 错误片段可以快速选中、循环和重练。
3. 实时音高、目标旋律、听辨、钢琴和错题复练存在于同一个离线应用。
4. 中文解释告诉用户“发生了什么以及如何复练”，不只给数字。
5. 无账号、断网可用、默认不上传声音。
6. 置信度不足时诚实显示“不足以判断”，不强行生成正式结论。
7. 未经教育校准的个性化顺序与题库只能称为练习建议，不冒充正式自适应评估。

## 12. 发布边界

本路线不降低 `docs/final-release-definition-of-done.md` 中的算法、真机、安全、可访问性和证据门槛。它将当前执行重心收窄到 Android 离线私测体验。Web 账号、私有云、OMR 和正式评分仍保留在长期正式版范围中，但不阻断当前本地 APK 私测候选版。
