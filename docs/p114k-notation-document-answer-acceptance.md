# P114k 五线谱 / 简谱文档答案活动验收标准

日期：2026-07-19

状态：implementation candidate
QA level recommendation：**strict**

## 1. 目标

P114k 补齐 `activity-answer` 已冻结但此前没有真实使用者的 `staff-notation` 与 `numbered-notation` 两种输入。真实入口必须来自 Stage E 中用户已检查、校验并明确确认的临时乐谱目标；系统把它冻结为当前会话内的 `score-document-v1` 确认修订，再由用户检查表示预览、主动提交并检查文档身份与修订一致性。

完整路径：

```text
手动草稿
→ 用户检查
→ 小节时值校验
→ 用户确认生成临时目标
→ 冻结确认谱面修订
→ 选择五线谱 / 简谱表示并检查预览
→ 主动提交
→ 非评分文档一致性证据
```

## 2. 必须交付

- `score-document-v1` 至少包含稳定文档标识、修订号、确认状态、来源指纹、拍号，以及 part / staff / voice / measure / event 层级。
- 只能从 `active` 且非空的确认临时目标创建文档；目标失效时失败关闭。
- 文档事件必须复制，不能与可变草稿共享对象引用。
- 五线谱答案使用 `staff-notation`，简谱答案使用 `numbered-notation`；活动只允许当前选择的输入方式。
- 切换表示方式必须重建本轮 session，不能保留另一输入方式的旧答案或检查证据。
- 提交必须包含当前文档标识与整数修订号；检查返回 `consistent`、`different` 或 `insufficient`，且始终为 `non-scoring`。
- 检查完成后可重置为新 attempt；清除或替换上游目标时，旧活动随上游状态失效或卸载。
- 用户可见文案必须是简体中文，并明确说明这是受控预览、会话内文档和非评分证据。

## 3. 表示边界

当前五线谱表示仅显示已确认事件的科学音高名、休止符与受控时值；当前简谱表示按限定的 C 大调音高范围显示 `1–7`、高音点、休止 `0` 和时值标记。它们是答案表示预览，不是完整雕版、拖拽制谱、MusicXML 编辑或 OMR 结果。

本切片不得宣称：

- 自动识谱、正式转写或出版级排版；
- 调号、临时记号、歌词、和弦、多声部、多谱表、连音或复杂时值支持；
- 用户演奏正确、课程通过、分数、等级或教育结论；
- 跨刷新保存、云同步、数据库持久化或可导出的正式谱面项目。

## 4. 状态与异常

| 场景 | 必须行为 |
| --- | --- |
| 上游目标为空 | 不挂载文档答案活动 |
| 上游目标 stale | 显示失效原因，不提供提交按钮 |
| 初始 ready | 显示预览，只提供主动提交 |
| 已提交 answering | 提供“检查文档与修订” |
| 已检查 checked | 显示非评分证据，提供重置 |
| 切换五线谱 / 简谱 | 清空旧答案和证据，创建新 session |
| 文档 id、revision 或 mode 不同 | 返回 `different`，不产生分数 |
| 没有可检查答案 | 返回 `insufficient` |

## 5. 自动门禁

- focused 纯逻辑测试：文档冻结、稳定身份、深复制、两种表示、stale 拒绝、活动定义、提交、检查与重置。
- 真实挂载 React 行为测试：两种表示切换、完整生命周期、重置和 stale 禁用。
- TypeScript、Lint、Next build、Android 本地静态校验与既有 activity/notation 回归必须通过。
- GitHub `quality`、`android-local` 和部署检查必须在合并前通过。

## 6. 手动 QA

Web `/practice` 至少执行一次：创建合法两小节草稿、检查、校验、确认目标、切换两种表示、提交、检查、重置、修改草稿使旧目标失效。当前 Android 离线入口没有挂载 Stage A–E 乐谱草稿流，因此本切片不宣称 APK 已有该入口；未来挂载到 APK 时必须另做触控与真机 QA。自动 DOM 测试和 CI 不等于浏览器视觉或真机 QA。
