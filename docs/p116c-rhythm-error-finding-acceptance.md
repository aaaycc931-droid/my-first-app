# P116c 节奏找错验收

最后更新：2026-07-20

P116c 是 P116 节奏专业套件的第三个最小完整纵向切片。它复用版本化节奏题库、Activity `choice` 协议、本机复练与学习画像，提供“看确认过的内置目标—听只含一处变化的版本—标记类型与位置—查看题内事件解释”的离线闭环。

## 用户闭环

1. Android 扩展节奏入口显示原始四四拍目标和 BPM。
2. 系统从题目版本与序号确定性地产生一处漏掉、拆分、合并或位移，并用本地 Web Audio 播放变化版本。
3. 完整播放结束前选项保持锁定；用户随后选择同时包含变化类型与拍内位置的标记。
4. 查看答案后只解释原目标与播放事件的可验证差异，并沿用现有最小复练目标与学习事实。

## 题目不变量

- 漏掉：从目标删除一个指定事件。
- 拆分：把一个指定事件替换为相邻的两个事件。
- 合并：把目标中距离最近的两个事件替换为其中点事件。
- 位移：保持事件数量，只移动一个指定事件，且不得静默等于原目标。
- 每题只有一个正确的“类型 + 位置”标记；四个候选 ID 唯一且由版本化输入确定。

## Fail-closed 与隐私边界

- 播放 token 绑定当前题目；重播、手动停止、全局停止、后台、重置与卸载都会清除完整播放资格、旧选择、计时器与脏 attempt。
- 迟到播放完成回调不能重新解锁已作废题目；本地音频准备失败时显示中文恢复原因。
- 不使用麦克风，不上传或保存声音；不新增账户、云端、数据库、存储 schema 或依赖。
- 不显示总分、百分比、等级、通过／失败，不从内置题目差异推断用户演奏能力。

## 自动验收

- focused domain tests：四种变换的事件数量／非等价不变量、候选唯一性、三档 Activity 与无评分字段。
- 真实挂载移动 React 行为：完整播放前锁定、播放后标记、题内解释、全局停止与迟到回调 fail closed。
- Android App 行为回归：扩展节奏入口真实挂载 P116a／P116b／P116c，旧选择题、复练与画像不回退。
- 无存储 schema 变化；旧复练队列与移动存储迁移测试继续通过。
- 合并前运行完整 `check`、Android sync／validator；GitHub `android-local` 是 API 36 Debug APK 权威自动门禁。

自动测试与 CI 不证明真实 Android 音频、真实用户辨认、教师审核或教育有效性。QA level recommendation：strict。
