# P111 音准专业基准协议与采集工具

更新：2026-07-18

QA level recommendation：**strict**

## 1. 交付结论

P111 冻结 `p111-pitch-benchmark-v1`。它为 P112 的多个录音后分析引擎提供同一输入和同一指标，复用 P104 的最低证据门槛，并增加专业扩展数据规模、voicing、RPA/RCA 和移动处理性能指标。当前仓库不包含真实录音、真实参与者记录或已通过的专业数据结果。

工具只读取被 Git 忽略的本地 JSON 汇总，不读取音频、不上传数据、不发起网络请求，也不生成正式分数、等级或通过/失败结论。

## 2. 冻结共同输入

每条样本必须具有：

- 唯一随机 `id`，以及预先确定的 `tuning`、`development` 或 `acceptance-blind`；
- `synthetic`、`instrument`、`real-voice` 或 `no-pitch` 来源类别；
- voiced/unvoiced 参考标签及 voiced 时的参考频率；
- 声区、强弱、稳定/颤音/滑音/气声、安静/环境噪声/扬声器泄漏和设备类别；
- 录音长度、引擎处理时间、预测 voiced 和预测频率；
- 真实人声的匿名参与者令牌、分开保存的授权记录令牌、基准使用许可、至少两名标注者和争议复核状态。

每次引擎运行必须冻结算法 commit、引擎 ID/版本、参数 SHA-256 和设备档案。不同引擎可以使用相同样本清单，但必须保存不同运行 ID 和参数摘要。

## 3. 盲测隔离

1. 在算法运行前分配三组；同一 ID 不得跨组重复。
2. `tuning` 只用于调参，`development` 用于开发比较，`acceptance-blind` 只用于最终盲测。
3. 盲测参考标签由未参与本轮调参的人封存；揭盲前设置 `labelsSealedBeforeRun=true`。
4. 引擎、模型、平滑、阈值和所有参数必须在揭盲前冻结，并记录参数 SHA-256；设置 `engineFrozenBeforeBlindRun=true`。
5. 看过盲测结果后发生的任何参数变化必须建立新版本和新盲测集，旧结果不得继续作为最终验收。

## 4. 指标定义

- voiced precision / recall / F1：以参考 voiced/unvoiced 和引擎 voiced/unvoiced 计算；无声、噪声与拒答样本不可删除。
- Raw Pitch Accuracy：参考 voiced 样本中预测 voiced 且绝对误差不超过 50 cents 的比例；漏检计失败。
- Raw Chroma Accuracy：忽略整八度后的误差不超过 50 cents；只用于暴露八度问题，不能替代 RPA。
- cents 中位数/P95：只对具有有效参考和有效预测的 voiced 对计算，同时必须与 voicing 指标一起报告，避免靠拒答美化误差。
- 八度错误率：参考 voiced 样本中，绝对误差至少 1150 cents 且折叠八度后不超过 50 cents 的比例。
- 实时系数 RTF：`processingMs / (durationSeconds × 1000)`；报告 P95。

所有指标必须同时按来源、声区、强弱、技巧、噪声和设备类别分层。总体均值不得掩盖气声、噪声或低/高声区失败。

## 5. 数量与退出门槛

P104 最低层继续要求：合成/乐器至少 200 条；真实人声至少 100 条、20 位参与者、4 类设备；覆盖低中高、轻中重、颤音和噪声；盲测真实人声绝对误差中位数不高于 35 cents、P95 不高于 100 cents、八度错误率不高于 2%。

专业扩展层目标：合成/乐器至少 1000 条；真实人声至少 300 条、40 位参与者、6 类设备；voiced F1 不低于 0.95；RPA ±50 cents 不低于 0.97；三档 Android 离线处理 RTF P95 不高于 0.5。三档 Android 必须分别报告，不能只填一个汇总设备档案。

数据数量或指标通过不等于正式评分获批。正式分数仍需 P119 教育校准、重复性、公平性、阈值审批和真实用户验证。

## 6. 本地采集流程

1. 复制 `local-fixtures/pitch-benchmark/benchmark.example.json` 为同目录 `benchmark.local.json`。
2. 按目录 README 建立匿名样本、授权和标注台账；原始录音保存在产品所有者控制的本地加密位置。
3. 先分组、封存盲测标签并冻结引擎，再运行引擎并填写结果。
4. 执行 `npm run validate:pitch-benchmark-protocol`。
5. `BLOCKED` 表示数据或证据尚缺，不是工具崩溃。全部通过后仍要人工核对授权原件、标注者独立性、揭盲日志和原始音频摘要。

## 7. 隐私与仓库边界

- 禁止在 JSON 中写姓名、电话、邮箱、账号、学校、住址、精确位置、设备序列号或其他直接身份信息。
- 真实录音、授权文件、标注者身份和本地结果不得提交 Git、上传聊天、进入 CI artifact 或打包进 APK。
- 仓库可接收协议、空模板、纯合成测试和不含身份信息的最终汇总报告；最终汇总上传仍需产品所有者逐文件授权。
- 普通练习录音默认不是训练数据或基准数据；只有单独明确授权后才能进入本地基准。

## 8. 自动测试边界

CI 只验证协议计算、盲测隔离、隐私标志、八度/voicing 指标和失败边界。示例模板故意保持未授权、未封存和数量不足，因此不可能被误报为真实证据。真实数据校验是显式本地命令，不进入常规 CI。
