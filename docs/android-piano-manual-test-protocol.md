# Android 钢琴真机测试协议

更新：2026-07-17

本协议用于产品所有者在真实 Android 设备上验证 P107–P110 钢琴音频、事件录制、可选 MIDI 与谱面学习。自动测试和 CI 不能替代本记录。

## 当前可执行测试

记录设备型号、Android 版本、System WebView 版本、输出方式和 APK SHA-256。蓝牙结果必须与手机扬声器/有线输出分开。

1. **首次发声**：强制停止应用后冷启动，断网进入钢琴页，以手机扬声器连续执行 10 次首次按键。记录无声、明显延迟或失败次数；需要毫秒值时使用高帧率外部录像或音频回环，主观描述不得伪装成 P95 数值。
2. **32 音压力**：进入“完整 88 键”，将媒体音量调低，点击“运行 32 音压力测试（2 秒）”。连续执行 10 轮，记录爆音、明显卡顿、音量失控、应用无响应和测试结束后的残音。
3. **残音矩阵**：按键/松键、延音开关、停止全部、切换紧凑/88 键、切换单/双排、改变键宽、改变移调、返回首页、锁屏/解锁、前后台切换各执行 20 轮。任何声音在全停或后台后继续存在均为失败。
4. **采样听感**：分别检查 A0–B2、C3–B5、C6–C8，使用轻/中/重触控。记录爆音、削波、突兀音量、明显“加速/减速”移调感、异常音高、样本加载后突然换音色以及降级提示。
5. **移动交互**：横竖屏下检查单排/双排滚动、38/46/54/64 px 键宽、科学音高/固定唱名/隐藏标签、±12 半音移调、快速定位、10 指触控和系统返回。
6. **节拍器**：分别以 30、72、120、240 BPM 启停 4/4 节拍器，每档至少 1 分钟；检查重拍区别、页面切后台后静音、返回后可重新启动，以及钢琴与节拍器同时发声时无残音。
7. **演奏事件录制**：录制包含单音、和弦、延音开关和移调的片段，停止保存后回放；确认不请求麦克风权限、不保存音频，回放音符、踏板与原移调一致。
8. **回放控制**：检查 50%/75%/100%/125%/150% 速度、C4 分界的低/高声部过滤、有效 A–B 循环，以及 B≤A 时的拒绝提示；回放中点击“停止全部”、切后台和删除记录后不得残音。
9. **本机存储边界**：强制停止并重启应用后确认记录仍存在；导出 JSON 并检查只含事件与时间字段；删除必须二次确认。存储不可用时应明确提示“仅保留到本页关闭前”，钢琴弹奏仍可使用。
10. **六种音色预设**：依次选择原始、明亮、柔和、近场练习、复古和清脆短音；每档检查轻/中/重力度、C2/C4/C6、延音和快速重复。记录差异是否可听、是否削波，以及切换后是否残留旧音。六档共用同一 Public Domain 采样，不应记录为六套独立乐器采样。
11. **USB MIDI（条件能力）**：使用支持 OTG 的手机、数据线/转接器和至少一台 MIDI 键盘。点击“连接 MIDI”，记录系统是否提供权限与设备列表；测试最轻/最重力度、同时和弦、通道消息、延音踏板、拔线、重新插入、切后台和切设备。若 System WebView 显示“不支持 Web MIDI”，记录为 `UNSUPPORTED`，屏幕钢琴必须继续可用；不得把该结果记为硬件 PASS。
12. **MusicXML 草稿**：选择一个本机 `.musicxml`/`.xml` 简单单声部文件；确认选择时无网络请求，草稿未确认前播放按钮不可用。移除一个音符后确认状态必须失效；重新检查确认后测试 30/72/180 BPM 播放、停止、切后台和清除。当前入口不支持图片、PDF、压缩 MXL 或完整 MusicXML 记谱。
13. **谱面/瀑布学习视图**：检查项目原创 8 音练习和已确认 MusicXML 草稿在谱面、瀑布两种视图下可读；横竖屏、360 CSS px 等效宽度、字体放大和屏幕阅读器名称不得阻断播放/停止/清除。

## 初代功能完整 APK 后执行

以下项目按产品所有者决定，放在 P118 功能完整的初代评估 APK 之后，不阻塞此前的运行时开发：低/中/高三档 Android 完整矩阵、2 名视唱练耳教师审核、5 名目标用户可用性测试，以及至少 100 段真实人声/20 位参与者及授权记录。它们仍是最终候选前的硬门槛，不能因执行顺序后移而删除。
