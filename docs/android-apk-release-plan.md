应用名称：`视唱练耳`

## 1. 当前产品决策

当前阶段只交付可直接安装、可由产品所有者私下传递的 Android APK，不进行应用商店上架准备。中国用户无法稳定访问 Web 入口，因此 APK 的核心练习必须随安装包一起提供，启动和使用不能依赖 Vercel、Supabase、生产域名或其他远程网页。该边界只约束当前私测载体；最终产品同时建设高质量本地与中国区合规云端能力，不能把离线私测误写成最终定位。

2026-07-22 远端已核对 P106–P113、P114a–P114m、P115a–P115i、P116a–P116d 与 P117a–P117e 合并；P117e 通过 PR #399 合并，P117 状态收尾通过 PR #400 合并。真实 Android、真实麦克风、真实人声与教育审核仍无闭环证据。当前 0.2.0 Debug APK 仍只可作为技术私测包。八产品范围包含制谱、伴奏和云端泳道，P120 只能是范围明确的专业私测候选，不是公开最终版。

当前明确不做：

- Google Play、国内应用商店或其他商店上架；
- AAB、商店截图、分级、商店文案、数据安全表或上架审核材料；
- TWA、PWA 安装、Digital Asset Links、App Links 或远程 `server.url`；
- 在本地私测 APK 中接入账号、云同步、私有上传或正式评分。

Web 产品继续保留并独立部署；它已有的 Supabase 账号、课程和私人记录能力不会被删除，但不是当前 APK 的启动依赖。

## 2. 选定架构

当前 Android 客户端使用 **Vite 静态移动入口 + Capacitor Android 本地 WebView**：

```text
Android APK
→ Capacitor WebView
→ APK 内的 mobile-dist 静态资源
→ 本地 React 练习组件与 Web Audio
```

硬性边界：

- `capacitor.config.ts` 只使用 `webDir: "mobile-dist"`，不得配置远程 `server.url`。
- 私测 APK 不声明 `android.permission.INTERNET`。
- 移动构建用本地桩替换 Supabase 客户端，不把生产 URL、publishable key 或会话逻辑打入移动 bundle。
- 题库、答案规则和声音由本地确定性逻辑生成。独立本地练习在每个应用会话生成一个仅在内存中的随机种子，并以该种子洗牌题序；相同种子和题号必须复现相同题目，完整一轮不得重复。课程练习保持固定种子，避免已保存的课程题号与服务端答案规则不一致；关闭页面或应用后清除种子和当前练习状态。
- 只有 Android 移动入口可以持久化本机错题复练队列；共享 Web `/practice` 和课程练习不得读写该队列。队列只保存复现题目所需的最小目标，并以稳定题目标识避免本地题库扩容后旧错题漂移；P115–P117 已接入活动的 schema/catalog 迁移保持冻结规则和 fail-closed 读取，P117d/P117e 继续保持 session-only 且不写复练队列。不保存原始作答、音频、分数、准确率、账户或云端学习历史，也不连接云端。
- Web 和 APK 可以复用经过测试的练习组件与算法，但有独立入口、配置和发布门槛。
- Android 显式启用版本化本地题库：既有基础题型和 P115–P117 活动保持稳定题目标识、难度与迁移边界；共享 Web 和系统课程继续使用既有契约，防止本地题库扩容改变云端 v1 或既有课程答案。

## 5. 构建基线

- Node.js：`>=22.12.0`
- Capacitor：8.4.2
- Vite：8.1.4
- JDK：21
- Gradle：8.14.3
- minSdk：24（Android 7）
- compileSdk / targetSdk：36
- Android Build Tools：35.0.0

`versionName` 与 `versionCode` 的唯一来源是根 `package.json` 的 `version` 与 `solfeggio.androidVersionCode`；`package-lock.json`、Gradle 和私测变更记录必须通过 `npm run validate:android-version` 保持一致。当前候选为 `0.2.0` / `2`。CI 使用的第三方 Actions 固定到经过官方仓库核对的完整 commit SHA，Gradle wrapper 同时固定 8.14.3 `-all` 分发摘要；`npm run test:ci-supply-chain-policy` 防止恢复成浮动 tag 或缺失 checksum。固定依赖不会自动吸收补丁，升级必须单独审查并重新跑完整门禁。

常用命令：

```bash
npm run mobile:build
npm run validate:android-local
npm run android:sync
npm run android:build:debug
npm run verify:android-local-artifact
```

Android SDK 与 JDK 属于本机工具，不提交仓库。`mobile-dist`、Gradle 输出和 APK 工件也不提交 Git；每次交付单独记录 commit、文件名、签名类型和 SHA-256。

GitHub Actions 的 `android-local` job 只在以下步骤全部成功后上传私测工件：依赖安装、静态移动 bundle 校验、`testDebugUnitTest`、`assembleDebug`、APK 签名验证、APK 包名/版本/SDK/权限检查，以及 APK 内本地资源与云端标识检查。下载的工件目录包含 debug APK、同名 `.sha256` 文件以及 JSON/Markdown 构建报告（版本、commit、workflow run 和边界检查结果）。保留 14 天；它是私测调试包，不是 AAB、专用 release 签名包或真实手机验收证据。

### CI 候选 APK 工件

Android CI 在完成本地 bundle 校验和 `assembleDebug` 后，应上传名为 `solfeggio-local-debug-apk-<commit>` 的可下载候选工件，且只包含一个调试 APK、其同一次构建生成的 SHA-256 文件以及 JSON/Markdown 构建报告。打包脚本之后、上传动作之前必须运行独立 verifier：要求目录精确包含四个普通文件并拒绝符号链接，重新计算 APK SHA-256 与字节数，核对 `.sha256` 的文件名/摘要、JSON 报告的文件名/摘要/字节数/版本/commit，以及 Markdown 的文件名/摘要/版本/commit；缺失、多余或不一致必须阻止上传。包名、SDK、签名、权限、本地资源和云端边界仍由此前的打包脚本使用 `aapt`、`apksigner` 与 APK 内容检查验证，独立 verifier 不重复声称这些检查。上传步骤不能上传空目录、沿用旧文件或绕过 verifier。

该工件仅供私下测试下载：它仍是 Android Debug 签名，既不是专用 release 包，也不证明真机安装、飞行模式、音频、后台恢复或 20 轮稳定性已通过。每次用于测试的下载物都必须在测试记录中写明 CI 运行、commit、文件名和校验和；可下载性本身不能替代签名、结构或真机验收证据。

GitHub artifact 页面显示的 digest 是下载 ZIP 容器的 SHA-256，不等同于 ZIP 内 APK 的 SHA-256。交付记录必须从 verifier 已核对的 `.sha256` / JSON 报告读取 APK 摘要，并同时记录 artifact ID 与 ZIP digest，避免把两个不同层级的摘要混用。P91 PR #342 run `29490109582` 的 `quality` 与 `android-local` 均 PASS；artifact `8372077214` 的内部 APK SHA-256 为 `c5060dad9fd6f11401b5c6eb2d08319b082272068a6abccb59dd20d819ecbcd4`，GitHub ZIP digest 为 `fd416cf87553ed3d44d96a50ef79f1d0b373700d6554368474a4b6d82366fe11`。该证据让 C1 达到 PASS，但不替代 0.2.0 新版本的本切片 CI，也不替代真机验收。

## 6. 权限、生命周期与数据

- APK 不请求网络或宽泛存储权限。实时音高页只有在用户明确点击“开始实时反馈”后才请求麦克风；会话录音复用这条已授权的本地媒体流且必须再次主动开始，普通听辨与参考钢琴不会触发权限。
- 权限拒绝、无设备或占用必须显示中文原因，并保留听辨与钢琴等非麦克风路径。
- 停止、切换练习、进入后台和组件卸载必须停止音源与计时器；恢复后不自动播放、录音或提交。
- 当前选择、答案显示状态、默认实时麦克风帧以及独立本地题序种子只存在于 WebView 内存。Android 有两项用户可控的本机持久化例外：最多 12 项的错题复练目标写入 `localStorage`；用户点击保存后，最多 20 条练声记录写入应用私有 IndexedDB，每条最多 600 帧、200 字备注、目标快照和可选 5 MB 录音。二者均不写账号或云端；练声 JSON 导出不含录音二进制。
- 复练队列不保存答案、用户选择、音频、分数、准确率、账户、完整历史或云端标识。共享 Web `/practice` 和课程练习不读写该队列；课程练习保留固定题序以保证与课程答案规则一致，且不会写入本机错题。所有模式均不在 APK 中同步云端记录。
- 卸载应用、清除应用数据或从 debug 签名切换到不兼容的专用 release 签名会删除本机复练与练声记录；JSON 只能备份曲线与摘要，不能备份录音。首次对外分发最终签名包前必须冻结签名/升级策略并完成覆盖升级验证。
- 后续如加入本地文件，默认使用系统文件选择器，不申请宽泛存储权限。

## 7. 真机私测清单

每台设备至少执行：

1. 从聊天或文件管理器安装 APK，确认系统显示应用名和图标。
2. 在飞行模式下冷启动，确认首页和七个练习入口可打开。
3. 每类练习完成播放、停止、选择、查看答案、错题复练、重置和下一题。
4. 每类至少答错一题，确认首页“本机复练”数量变化、错题置顶且可复现；随后答对该题，确认它从队列移除。关闭并重启应用后确认未清除的队列仍在。
5. 在清除复练时先取消二次确认，确认列表不变；再次清除并确认，确认列表归零。该检查不得替代卸载或清除应用数据后的丢失边界验证。
6. 分别从独立页和每类练习内展开本地参考钢琴，验证三档音域、单击、长按、至少双指两键、音量、延音开关、停止全部和错误重试；收起钢琴后不得残留声音。
7. 快速松开、触控滑出、系统手势打断、切换音域、切换题目声音与钢琴声音，确认没有延迟后突然发声、叠音或卡住的按下状态。
8. 验证麦克风拒绝与允许、实时曲线、目标同步、录音/停止/回放/丢弃；参考音、会话录音与已保存录音不得和监听互相串音。
9. 保存一条含曲线和录音的练声记录，重启后回看/回放；导出 JSON 并确认没有录音二进制；验证单删、清空取消/确认、20 条/5 MB 上限和存储失败中文降级。
10. 播放或监听中切到后台、锁屏或切换应用再返回，确认媒体轨、录音与声音停止，返回后不得自动恢复。
11. 连续完成 20 轮题目播放、钢琴、实时曲线、录音、保存/删除和页面切换，不出现卡死、残音、持续变慢或可复现内存增长。
12. 记录手机品牌型号、Android 版本、系统 WebView 版本、结果和问题截图。

如果手机提示“未知来源”，只允许本次使用的文件管理器或聊天工具安装未知应用；安装完成后可以立即关闭该授权。不要关闭 Play Protect 或系统安全扫描。

## 8. 完成判定

当前 APK 私测里程碑完成需要：

- 本地构建、校验、签名和安装包结构全部通过；
- 产品所有者至少在一台真实手机完成第 7 节首轮清单；
- 没有阻断安装、启动、音频和核心练习的 P0/P1 缺陷；
- 已知问题和下一测试版本有记录。

最终私下正式版本还需要专用 release 签名、覆盖升级、回滚和三档真实 Android 设备证据。应用商店准备不属于当前完成条件，只有产品所有者以后明确恢复时才重新规划。
