# Android 本地 APK 私测路线

状态：**当前最高优先级 / 私下分发测试**
生效日期：2026-07-16
应用 ID：`com.aaaycc931.solfeggio`
应用名称：`视唱练耳`

## 1. 当前产品决策

当前只交付可直接安装、可由产品所有者私下传递的 Android APK，不进行应用商店上架准备。中国用户无法稳定访问 Web 入口，因此 APK 的核心练习必须随安装包一起提供，启动和使用不能依赖 Vercel、Supabase、生产域名或其他远程网页。

当前明确不做：

- Google Play、国内应用商店或其他商店上架；
- AAB、商店截图、分级、商店文案、数据安全表或上架审核材料；
- TWA、PWA 安装、Digital Asset Links、App Links 或远程 `server.url`；
- 在本地私测 APK 中接入账号、云同步、私有上传或正式评分。

Web 产品继续保留并独立部署；它已有的 Supabase 账号、课程和私人记录能力不会被删除，但不是当前 APK 的启动依赖。

## 2. 选定架构

当前 Android 客户端使用 **Vite 静态移动入口 + Capacitor Android 本地 WebView**：

```text
Android APK
→ Capacitor WebView
→ APK 内的 mobile-dist 静态资源
→ 本地 React 练习组件与 Web Audio
```

硬性边界：

- `capacitor.config.ts` 只使用 `webDir: "mobile-dist"`，不得配置远程 `server.url`。
- 私测 APK 不声明 `android.permission.INTERNET`。
- 移动构建用本地桩替换 Supabase 客户端，不把生产 URL、publishable key 或会话逻辑打入移动 bundle。
- 题目、答案和声音由本地确定性逻辑生成；关闭页面或应用后清除当前练习状态。
- Web 和 APK 可以复用经过测试的练习组件与算法，但有独立入口、配置和发布门槛。

## 3. 当前已交付的私测切片

- 本地首页和底部练习导航；
- 单音听辨；
- 上行/下行音程听辨；
- 四拍节奏听辨；
- 三音旋律听写；
- 本地 Web Audio 播放、停止、重置、下一题和错题复练；
- 统一的本地音频启动、播放、停止与定时器清理；音频只能在用户点击后恢复；
- 应用切换练习、进入后台、页面冻结、页面卸载或 Android 原生暂停时停止声音并暂停音频上下文；恢复时明确提示题目已重置，不自动播放；
- Android System WebView 兼容构建目标、启动失败中文降级页和运行时异常恢复页；
- 中文应用名、项目图标、启动画面和安全区域布局；
- 固定包名、versionCode、versionName、本地 bundle 校验和调试 APK 构建。

这些能力只提供答案说明和非评分练习反馈，不提供正式成绩、准确率、等级或通过/失败判断。

## 4. 交付阶段与状态

| 阶段 | 交付物 | 当前状态 | 退出条件 |
| --- | --- | --- | --- |
| A. 本地移动入口 | Vite 入口、四类练习、本地桩、相对资源路径 | PASS | 静态构建通过；无生产云端标识；断网边界可自动校验 |
| B. Android 工程 | Capacitor 工程、固定包名、图标、启动画面、无网络权限 | PASS | Gradle 构建成功；APK 结构、包名、SDK 和权限检查通过 |
| C. 私测 APK | V2 调试签名 APK、SHA-256 | PASS | APK 可下载；签名验证通过；校验和已记录 |
| D. 真机验收 | 安装、启动、四类播放/停止/答案/重置、后台恢复 | IN_PROGRESS | 至少一台真实 Android 手机完成首轮测试；最终候选覆盖三档设备 |
| E. 私下正式包 | 专用 release key、版本升级和回滚包 | NOT_STARTED | 功能冻结后再创建；密钥不入仓库；安装、覆盖升级和回滚经过验证 |

调试签名 APK 是可安装的私测包，不得冒充最终私下正式包。当前先让真实用户测试功能，再决定专用 release key 和最终版本号。

当前候选包的哈希、签名、结构检查和未执行项记录在 `docs/android-private-test-build-evidence.md`。

## 5. 构建基线

- Node.js：`>=22.12.0`
- Capacitor：8.4.2
- Vite：8.1.4
- JDK：21
- Gradle：8.14.3
- minSdk：24（Android 7）
- compileSdk / targetSdk：36
- Android Build Tools：35.0.0

常用命令：

```bash
npm run mobile:build
npm run validate:android-local
npm run android:sync
npm run android:build:debug
```

Android SDK 与 JDK 属于本机工具，不提交仓库。`mobile-dist`、Gradle 输出和 APK 工件也不提交 Git；每次交付单独记录 commit、文件名、签名类型和 SHA-256。

## 6. 权限、生命周期与数据

- 当前听辨切片只播放本地合成音，不请求麦克风、存储或网络权限。
- 后续只有在用户明确点击录音时才能请求麦克风；拒绝后必须显示中文原因和非录音路径。
- 停止、切换练习、进入后台和组件卸载必须停止音源与计时器；恢复后不自动播放、录音或提交。
- 当前练习状态只存在于 WebView 内存，不写账号、数据库、localStorage 或 IndexedDB。
- 后续如加入本地文件，默认使用系统文件选择器，不申请宽泛存储权限。

## 7. 真机私测清单

每台设备至少执行：

1. 从聊天或文件管理器安装 APK，确认系统显示应用名和图标。
2. 在飞行模式下冷启动，确认首页和四个练习入口可打开。
3. 每类练习完成播放、停止、选择、查看答案、错题复练、重置和下一题。
4. 播放中切到后台、锁屏或切换到其他应用再返回，确认没有残留声音且出现重置提示；返回后声音不得自动恢复。
5. 连续完成 20 轮播放和切换，不出现卡死、重复声音或持续变慢。
6. 记录手机品牌型号、Android 版本、系统 WebView 版本、结果和问题截图。

如果手机提示“未知来源”，只允许本次使用的文件管理器或聊天工具安装未知应用；安装完成后可以立即关闭该授权。不要关闭 Play Protect 或系统安全扫描。

## 8. 完成判定

当前 APK 私测里程碑完成需要：

- 本地构建、校验、签名和安装包结构全部通过；
- 产品所有者至少在一台真实手机完成第 7 节首轮清单；
- 没有阻断安装、启动、音频和核心练习的 P0/P1 缺陷；
- 已知问题和下一测试版本有记录。

最终私下正式版本还需要专用 release 签名、覆盖升级、回滚和三档真实 Android 设备证据。应用商店准备不属于当前完成条件，只有产品所有者以后明确恢复时才重新规划。
