# Stage D：手动乐谱草稿小节时值校验验收标准

## 1. 阶段目标

Stage D 接收 Stage B 的当前会话内手动乐谱片段草稿，在用户完成检查后，按小节汇总事件时值并与当前拍号比较。它只建立未来 Stage E 之前的可解释校验门槛，不创建练习目标。

## 2. 输入与计算范围

- 输入必须是当前 Stage B 草稿，包含固定两个小节。
- 支持既有 `2/4`、`3/4`、`4/4` 拍号，以及二分、四分、八分音符和四分休止符。
- 二分、四分、八分时值分别按 `2`、`1`、`0.5` 个四分音符拍计算；休止符与同值音符等量计入。
- 每个小节独立比较。必须展示应有时值、实际时值、相差时值，以及“时值完整 / 时值不足 / 时值超出”。
- 只有两个小节都恰好符合拍号时，当前校验结果才通过。

## 3. 运行门槛与状态

- 空、已清空、无效、未检查或 stale 草稿不能运行校验，并显示简体中文原因。
- 用户标记当前草稿为已检查后才能主动运行校验。
- 状态至少包括：无结果、不可运行、未通过、通过、旧结果已失效、已清除。
- 修改或删除事件、添加事件、改变拍号、取消检查、来源替换或清除、转为独立草稿、重新确认来源、清空或重置草稿，都必须使旧校验结果失效或清除。
- 清除校验结果不能清除 Stage A 图片、Stage B 手动草稿或 Stage C 模拟草稿。

## 4. 交互与文案

- Stage D 面板位于 Stage B 手动草稿之后。
- 支持运行、重新运行和清除校验结果。
- 用户可逐小节看到校验依据，不只看到一个总体结论。
- 用户可见文案默认使用简体中文；`Stage D`、`Stage E`、`/api/recognize`、`localStorage`、`IndexedDB` 等必要技术标识可以保留。
- 不使用正确率、分数、识谱成功、正式谱面、最终目标等会误导为正式结果的表达。

## 5. 阶段边界

- 校验通过只表示小节时值总和符合当前拍号，不表示音高正确、识谱正确、图片内容被读取、草稿成为正式谱面或演唱可以评分。
- Stage D 不实现真实 OCR / OMR、图片分析、PDF 解析、Audiveris production runtime、上传、`/api/recognize`、账号、数据库或持久化。
- Stage D 不创建临时或最终练习目标，不开放进入练习；Stage E 入口保持禁用并显示原因。
- 草稿与校验结果只存在于当前页面内存，刷新后消失，不写入 `localStorage` 或 `IndexedDB`。

## 6. 自动测试最低覆盖

- 空草稿、未检查草稿和 stale 草稿不能校验。
- `2/4`、`3/4`、`4/4` 的预期拍数计算正确。
- 二分、四分、八分音符和四分休止符换算正确。
- 两小节分别得到完整、不足或超出结果，总体状态正确。
- 修改事件、改变拍号、来源失效、清空草稿后旧结果失效。
- 校验通过只暴露未来 Stage E 就绪状态，不创建练习目标。

## 7. QA 建议

QA level recommendation：strict。Stage D 新增状态化校验门槛，并与 Stage B 的编辑、检查、来源 stale、清空和重置流程联动；合并前必须通过 focused test、现有 Stage B / Stage C 回归、生产构建和浏览器交互 QA。
