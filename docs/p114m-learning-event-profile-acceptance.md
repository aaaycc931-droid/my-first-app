# P114m：本机学习事件与非评分画像验收标准

状态：**IMPLEMENTATION CANDIDATE / 等待远端 PR 与 CI**  
日期：2026-07-19  
QA level recommendation：**strict**

## 1. 目标

以 Android 现有四类本地练耳和本机复练为真实使用者，冻结最小 `learning-event-v1` 与 `learning-profile-v1`。它们只表达用户已经完成的练习事实、错误、复练动作和建议原因，不表达正式能力评级，也不与伙伴关系数据混合。

## 2. 真实使用路径

1. 用户在单音、音程、节奏或旋律听写中主动查看答案后，记录一次 `result-checked`。
2. 事件只包含题型、难度、稳定目标键、随机/复练模式、正确/错误、顺序和本机时间。
3. 用户从复练队列或画像建议进入某题时，记录一次 `review-started`。
4. 画像从这些事件更新已核对、正确、错误、开始复练和复练答对次数，并按题型保留同类事实。
5. 建议只选择现有复练队列中的真实题目，并显示“为什么建议”；关闭建议后核心练习与复练队列保持可用。
6. 用户可二次确认后单独清空画像与事件；复练队列不随之删除。

## 3. 数据与隐私边界

- 最多保留 48 个近期事件，画像计数独立累积；序列化上限为 24 KiB。
- 不保存用户选择、正确答案内容、音频、录音、邮箱、账号、原始证据或伙伴关系。
- 不上传、不联网、不写 Supabase；卸载或清除应用数据后消失。
- 画像与复练队列使用不同 storage key，可分别关闭、清除和故障降级。
- 解析严格检查 schema、精确字段、计数一致性、事件顺序和容量；损坏记录失败关闭并清除。

## 4. 教学与文案边界

- UI 必须明确显示“本机事实，不是能力评分”。
- 不生成分数、等级、准确率、通过/失败、能力标签或薄弱能力结论。
- 建议原因只陈述已核对错误次数与当前待复练题数，不作能力推断。
- 本切片不实现正式自适应、跨设备同步、教师画像、伙伴成长或生成式 AI。

## 5. 状态与故障

- 空画像：显示全部计数为零，且没有可建议题。
- 建议关闭：显示关闭状态，练习与复练不受影响。
- storage 不可用、读取、保存或清除失败：显示中文提示且不阻断练习。
- 重置：必须二次确认；失败时保留当前内存状态，不伪称已清除。
- 清空画像不得清除现有复练题；清空复练题也不得暗中清空画像。
