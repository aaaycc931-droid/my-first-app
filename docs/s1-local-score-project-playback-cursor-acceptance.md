# S1 本机谱项目播放光标切片验收

## 定位

本切片在本机谱项目的保存、编辑、播放和图形五线谱基础上继续交付：

> 播放当前已保存 revision → 按事件完整时值推进光标 → 休止符也可见 → 停止或生命周期中断时立即清除。

它是 S1 播放检查路径的 implementation candidate，不代表节拍器、循环、完整排版、教育审核或推广测试已经完成。

## 本批范围

- 播放计划为每个被选声部的音符和休止符生成确定性时间跨度。
- 时间跨度绑定 document ID、revision、part、staff、voice、measure 和 source event ID。
- 音符光标保持完整记谱时值；采样音频仍按既有 gate 提前释放，两者不混为同一语义。
- 休止符没有音频 note-on，但在其完整持续时间内成为当前活动事件。
- 同时开始的多声部事件可并发处于活动状态，顺序保持确定。
- 五线谱预览仅显示第一声部；隐藏声部的活动 ID 不会伪造可见 token。
- 当前活动 token 使用既有琥珀色播放标记，蓝色选择框继续独立存在。
- 主动停止、全局音频停止、计划结束、revision 变化、失焦／后台清理和组件卸载都会取消未来计时器；旧光标不得复活。

## 自动验收

1. 计划测试覆盖音符、休止符、尾部休止、rest-only 声部、并发声部、BPM 限制、超拍阻断和确定性 schedule。
2. Hook 行为测试覆盖起始音符、音频释放后的剩余光标、休止、下一音符、自然结束、主动停止、全局停止、revision 变化和卸载。
3. 五线谱组件测试覆盖选择与播放标记独立、完整 aria 状态以及点击／键盘选择。
4. 既有面板保存、第二小节编辑、撤销／重做和重开测试继续通过。
5. Quality 必须覆盖全部 `test:*` 脚本且每项恰好一次；lint、typecheck、移动端构建、Next.js 构建和 Android 本地 CI 继续通过。

## 严格人工 QA

QA level recommendation：**strict**。本切片直接修改音频播放调度期间的 UI 状态与清理路径。

在真实浏览器和安装后的 Android WebView 上分别执行：

1. 创建包含“音符 → 休止 → 音符”的谱，播放并确认琥珀光标按完整记谱时值依次移动。
2. 确认第一个音符声音释放后、休止开始前，光标仍保持到该音符时值结束。
3. 选择一个事件后播放，确认蓝色选择框和琥珀播放标记可同时辨认。
4. 播放中主动停止、切换页面、失焦、锁屏／进后台并恢复，确认声音和光标立即清除且不会自行恢复。
5. 播放中修改项目形成新 revision，确认旧计划、旧声音和旧光标全部失效。
6. 在多小节与横向滚动状态下确认当前 token 可见；屏幕阅读器应播报“正在播放”。

没有真实执行记录时，上述浏览器、Android WebView 和物理设备 QA 必须保持 `NOT_EXECUTED`。
