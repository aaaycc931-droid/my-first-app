# P53 — 临时乐谱视唱本地跟练反馈验收标准

## 1. 阶段目标

P53 在既有 Stage E 临时视唱练习目标与既有浏览器本地录音 / 音高估计之间增加一个明确、可撤回的非评分反馈闭环。它只服务于用户主动选中的当前视唱音符，不创建正式目标、正式转写或成绩。

## 2. 进入条件与用户流程

- 用户先完成 Stage B 手动草稿检查、Stage D 小节时值校验与 Stage E 临时视唱目标确认。
- 在“临时乐谱练习”中，只有当前非休止的视唱音符显示“使用当前音符进行本地跟练”。
- 点击后切换到既有“练习反馈”功能区，并明确显示目标音符、事件序号和参考频率。
- 用户仍需主动开始本地录音、停止录音并点击“进行本地音高估计”；P53 不自动请求麦克风、不自动录音或自动分析。
- 估计完成后，才显示当前事件的本地非评分音高提示；用户可以返回临时乐谱练习继续下一个事件。

## 3. 反馈与失效语义

- 反馈只使用当前 Stage B 支持范围内的视唱音符频率和最新浏览器本地音高估计。
- 提示可以为“接近目标音”“略偏高，可以稍微放低一点”“略偏低，可以稍微抬高一点”或“没有检测到稳定音高”。
- 反馈必须明确是本地、非评分参考，不产生等级、通过 / 失败、正式成绩或综合视唱评测。
- 音高估计在选中当前事件之前取得、在选中后切换了目标，或关联目标 / 事件不再当前时，必须显示为不能用于当前事件，要求重新估计。
- 草稿、校验变化导致临时目标 stale，或用户清除 / 新建临时目标时，当前乐谱跟练上下文必须失效；旧结果不得继续作为当前事件反馈。

## 4. 不适用事件与会话边界

- 休止事件显示不需要音高跟练的说明，不进入音高反馈。
- 临时节奏目标不提供本阶段的音高跟练入口。
- 继续使用既有浏览器本地录音与音高估计；不新增上传、云端、账号、数据库、localStorage、IndexedDB 或持久化练习记录。
- 刷新后临时目标、乐谱跟练上下文、录音和估计结果全部消失。

## 5. 禁止事项

- 不实现 OCR / OMR、图片像素分析、PDF 解析、Audiveris、`/api/recognize`、上传、真实识谱结果、最终目标、正式转写、正式评分或成绩。
- 不替换既有“本地旋律”流程，不新增第二套录音、播放、钢琴或音高估计 runtime，不新增依赖。

## 6. 最低自动测试

- C4 至 C5 支持范围映射到稳定的参考频率，休止 / 无音高无目标频率。
- 覆盖无目标、无可靠估计、接近、偏高和偏低提示，以及音分计算。
- Stage E 既有目标创建 / stale 回归继续通过。

## 7. QA 建议

QA level: strict。需要 focused tests、Stage B / C / D / E 回归、生产 build、远端 PR checks，以及合并后公开生产站点的浏览器交互 QA。浏览器 QA 至少验证：当前视唱音符能主动进入反馈区；旧估计显示 stale；休止和节奏目标无音高入口；清除 / stale 目标后上下文消失；不触发上传或 `/api/recognize`。
