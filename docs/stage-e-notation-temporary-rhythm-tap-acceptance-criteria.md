# P54 — 临时乐谱节奏拍击练习验收标准

P54 为已确认的 Stage E **临时节奏练习目标**增加一个 browser-local、session-only 的点击练习闭环。它把手动草稿中每个非休止事件的起始位置转换为 tap 参考点，并复用现有浏览器节拍器、预备拍、点击输入与非评分节奏反馈。

## 前置条件

- 用户已经完成 Stage B 手动草稿检查和 Stage D 小节时值校验。
- 用户在 Stage E 明确创建了当前有效的“临时节奏练习目标”。
- P54 只接受该 active temporary rhythm target；视唱目标、休止事件和 stale target 不进入此流程。

## 用户流程

1. 用户在“临时乐谱练习”中进入一个有效的临时节奏目标。
2. 用户明确点击“使用此临时节奏目标进行拍击练习”。
3. 页面切换到既有“节拍与节奏”功能区，并显示该临时目标来源、拍号、非休止事件数与当前会话边界。
4. 用户主动点击开始；现有节拍器可播放预备拍，随后按草稿事件起点进行 tap 或按空格键。
5. 页面只显示接近、偏早、偏晚、漏掉、额外等非评分提示，以及可选的当前会话点击延迟校准影响。
6. 用户可停止、重置或返回临时乐谱练习；这些操作不修改草稿、校验或目标本身。

## 状态与失效

- 目标事件起点根据草稿事件顺序和二分 / 四分 / 八分时值计算；休止只推进时间，不要求 tap。
- 目标变为 stale、被清除、被重新创建，或其 id / fingerprint 不再匹配时，当前节奏练习上下文、目标点与 tap 记录必须被清除并停止节拍器。
- 切换普通节奏练习时，临时节奏上下文不应被误用；普通四分 / 八分脉冲行为保持不变。
- 练习状态刷新后消失，不写入 localStorage、IndexedDB、数据库或练习历史。

## 明确边界

- 不进行正式节奏评分、准确率百分比、等级、通过 / 失败、最终评测或正式成绩。
- 不使用麦克风起音作为 P54 输入；只使用用户主动 tap / 空格键的浏览器本地时间戳。
- 不读取图片像素，不实现 OCR / OMR、PDF parser、Audiveris 或 `/api/recognize`。
- 不上传、不调用云端 / AI API、不新增账号、数据库、持久化、依赖或第二套节拍器 runtime。

## 验收与 QA

- focused helper tests 覆盖时值到拍数、休止推进但不生成 tap 目标、不同 BPM 时间映射、空目标与非评分字段边界。
- 回归既有节奏 tap feedback、Stage B / D / E target tests、P53 pitch feedback test、build 与 `git diff --check`。
- 浏览器 QA 至少验证：有效 rhythm target 可进入本地拍击练习；预备拍后 tap 可产生非评分反馈；休止不要求 tap；停止 / 重置正常；目标 stale / clear 后上下文消失；普通节奏练习仍可用；刷新后状态消失；Network 无上传或 `/api/recognize`。

QA level：strict。
