## 1. Product identity

This project is a sheet music recognition MVP. Its purpose is to validate a minimal product flow for recognizing sheet music and previewing the resulting notes. It is not a portfolio page.

## 产品原则：用户便捷交互体验第一优先级

用户便捷的交互体验是本项目的第一优先级。项目最终不是为了展示技术链路，而是为了让中文用户能够低理解成本、低操作负担地完成视唱练耳练习。

未来所有功能设计都必须优先满足：

1. 操作步骤尽可能少。
2. 用户不需要理解 JSON、诊断数据结构、内部类型名或开发者概念。
3. 用户不应该在多个页面之间手动复制复杂数据，除非这是开发/调试备用入口。
4. 主流程应尽量一键完成，例如「发送到练习页预览」「开始练习」「清除导入预览」。
5. 用户可见文案应面向中文用户，当前阶段优先使用中文，不需要中英双语。
6. 所有研究性边界说明也要用中文表达，例如：这是研究预览，不是正式评分；不替换当前练习旋律；不上传音频；不写入练习历史；不参与音高评分。
7. 技术安全边界不能以牺牲用户体验为代价。JSON copy/paste 可以保留为开发/调试 fallback，但不能作为普通用户主路径。
8. 每个未来 PR 都应检查是否增加了用户操作负担；如果增加了，必须说明原因，并优先考虑更简单的交互方案。

## P64 — Temporary Notation Reference-Tone Playback (2026-07-13)

P64 adds one voluntary local reference aid to an active Stage E temporary sight-singing target: when the current event is a note, the user can click “播放当前参考音” to hear a short browser-local sine-wave reference tone for that event. The control is unavailable for rests and temporary rhythm targets. Playback is never automatic, does not request microphone access, does not record, does not create a score or completion result, and does not alter the manual round marker, draft, validation or temporary target.

The panel stops its own short reference tone when the target changes or the panel unmounts. It reuses the existing supported notation-pitch frequency mapping and exposes a Simplified Chinese recovery message if the browser cannot create audio. No upload, API call, localStorage, IndexedDB, account, database, cloud, OCR / OMR, final target, official transcription, dependency or scoring behavior is added. QA level recommendation: strict; type check, build, existing Stage E / P53 focused regressions, Vercel checks and post-merge production browser QA must cover voluntary playback, rest/rhythm absence, panel navigation or clear stopping playback, refresh and no-persistence boundaries.
