# P17a Complete Product Vision + Piano System Roadmap

> 历史状态说明：本文件保留 P17a 时点的长期规划和当时能力盘点，其中“当前已经/尚未实现”的表述不再代表最新生产状态。当前事实见 `docs/mvp-status.md`，正式版 V1 范围和退出条件见 `docs/final-release-definition-of-done.md`，实时差距见 `docs/final-release-status-matrix.md`。

## 1. 产品上线原则

本项目的长期上线原则如下：

1. 不把半成品包装成正式软件。
2. 不急于对普通用户上线。
3. 当前 P16 只能算 internal research prototype / internal alpha，不是完整用户产品。
4. MVP 只是最低可用目标，不等于正式上线标准。
5. 正式上线前，应让未来核心功能大部分真实实现。
6. 技术链路打通不等于产品可用。
7. 能演示不等于能上线。
8. 用户便捷交互体验是第一优先级。
9. 上线版本必须让用户能完成真实学习或练习闭环，而不是只看到诊断预览。
10. 只有当视唱、练耳、钢琴辅助、录音反馈、练习流程、中文体验和 QA 大体成熟后，才考虑面向用户开放。

## 2. 长期产品定位

项目长期定位为：**中文视唱练耳 + 钢琴辅助学习平台**。

长期模块包括：

1. 五线谱识别 / OMR。
2. 本地音频导入与旋律目标生成。
3. 音高练习与音高反馈。
4. 节奏练习与节奏反馈。
5. 视唱综合练习。
6. 练耳课程。
7. 钢琴系统。
8. 钢琴伴奏系统。
9. 录音与回放系统。
10. 学习进度追踪。
11. 中文用户体验系统。
12. 移动端 / APK / WebView 适配。

## 3. 当前已经真实实现的能力

截至 P16n，当前已经真实实现的能力包括：

1. 本地 WAV 选择。
2. 本地 decode metadata。
3. 音高帧诊断。
4. 类音符片段诊断。
5. 研究用目标音高曲线诊断。
6. 一键发送到 `/practice`。
7. `sessionStorage` browser-local handoff。
8. `/practice` 显示本地导入预览。
9. 清除导入预览。
10. 首页、`/research`、P16 handoff / preview 主要中文化。
11. Network panel QA 未观察到 upload / cloud / AI / API / account / database。
12. 原 Practice mock melody flow 保持可用。

## 4. 当前还没有真实实现的核心能力

当前仍缺失的核心能力包括：

1. 导入目标还只是只读预览。
2. 导入目标不能作为真实练习目标。
3. 用户不能基于导入片段真正练习。
4. 用户录音还没有和导入目标曲线做真实对比。
5. 没有非评分型反馈。
6. 没有正式评分。
7. 没有节奏评估。
8. 没有视唱综合评测。
9. 没有完整练耳课程。
10. 没有学习进度追踪。
11. 首页 OMR 仍是 mock recognition flow，不是正式识谱产品。
12. 没有 Song Learning Mode。
13. 没有钢琴组件。
14. 没有独立钢琴页面。
15. 没有音色系统。
16. 没有钢琴学习模式。
17. 没有钢琴录音管理。
18. 没有钢琴伴奏系统。
19. 没有 MIDI 外设支持。
20. 没有 APK / WebView 验证。
21. 没有完整 release QA。

## 5. Piano System 作为长期核心模块

Piano System / 钢琴系统是项目长期核心模块，不是附属小工具。

原因包括：

1. 视唱练耳学习离不开钢琴。
2. 用户需要钢琴来找音、听目标音、校准音高。
3. 教师和学生常用钢琴辅助视唱、练耳、节奏和旋律训练。
4. 未来伴奏应主要以钢琴为基础。
5. 钢琴组件可以连接 Practice Mode、练耳课程、音高反馈、伴奏系统和录音系统。

## 6. 练习界面内嵌钢琴组件规划

未来 `/practice` 应加入钢琴调用按钮，例如：

- 打开钢琴
- 收起钢琴

功能规划：

1. 在练习界面中调用底部或浮层钢琴键盘。
2. 可播放当前目标音。
3. 可辅助用户找音。
4. 可用于练习前定调。
5. 可配合导入目标片段。
6. 可配合音高反馈。
7. 不应遮挡核心练习流程。
8. 移动端需要适配横屏 / 竖屏。
9. 需要低延迟音频播放。
10. 需要中文按钮和说明。

## 7. 独立钢琴页面规划

未来可以新增独立钢琴页面，例如：`/piano`。

该页面长期目标：

1. 横向钢琴键盘。
2. 可滚动音域。
3. C4 / D4 / E4 等音名显示。
4. 中央 C 标记。
5. 单排键盘。
6. 双排键盘，后期。
7. 键宽调整。
8. 八度切换。
9. 音量控制。
10. 延音开关。
11. 节拍器。
12. 录音按钮。
13. 回放按钮。
14. 中文界面。
15. 未来连接练习页和录音管理。

## 8. 音色系统规划

可以参考“完美钢琴”这类软件的音色插件思路，但不能复制 UI、素材、图标、曲库或受版权保护内容。

早期音色可以包括：

1. 默认钢琴。
2. 明亮钢琴。
3. 电钢琴。
4. 木琴。
5. 管风琴。
6. 弦乐垫。
7. 合成音。

技术路线说明：

1. 早期可以先使用轻量合成音色。
2. 后期再考虑采样音色。
3. 采样音色需要考虑文件体积、加载速度、授权和版权。
4. 不引入未经授权的第三方音色资源。
5. 不复制外部应用的音色包。

## 9. 学习模式规划

未来钢琴学习模式可以包括：

1. 单音练习。
2. 音阶练习。
3. 音程练习。
4. 和弦练习。
5. 旋律跟弹。
6. 视唱跟练。
7. 听辨训练。
8. 简单歌曲片段练习。
9. 与 Practice Mode 连接。
10. 与练耳课程连接。

早期不做完整曲库，不做大型在线歌曲库。

## 10. 录音管理规划

未来录音管理可以包括：

1. 练习录音。
2. 钢琴录音。
3. 导入音频记录。
4. 本地回放。
5. 删除录音。
6. 重命名录音。
7. 标记练习来源。
8. 与某个练习片段关联。
9. 后期再考虑导出。
10. 后期再考虑账号和云同步。

早期必须保持：

1. 不上传。
2. 不写账号。
3. 不写数据库。
4. 用户可清除。
5. 隐私优先。

## 11. MIDI 外设规划

长期可以支持：

1. USB MIDI 键盘。
2. MIDI 输入检测。
3. MIDI 按键显示。
4. MIDI 作为练习输入。
5. MIDI 录音。
6. MIDI 回放。
7. 与钢琴页面连接。
8. 与练耳课程连接。

MIDI 外设支持需要单独规划，因为浏览器、手机、WebView、权限和兼容性都复杂。不能提前声明可用。

## 12. 钢琴伴奏系统规划

未来钢琴伴奏系统可以包括：

1. 单音目标音播放。
2. 和弦提示。
3. 简单和声进行。
4. 节拍器伴奏。
5. 钢琴分解和弦伴奏。
6. I-IV-V-I 基础和声。
7. 根据练习目标生成简单伴奏。
8. 导入旋律后生成辅助伴奏，后期。
9. 与视唱练习结合。
10. 与练耳课程结合。

## 13. 不能复制外部应用

本项目对外部应用的边界如下：

1. 可以参考“完美钢琴”这类软件的功能类型和用户需求。
2. 不能复制其 UI 布局。
3. 不能复制其图标。
4. 不能复制其图片素材。
5. 不能复制其音色资源。
6. 不能复制其曲库。
7. 不能复制其品牌表达。
8. 本项目必须设计自己的中文学习体验和交互结构。

## 14. 长期路线建议

### P17 — Complete Product Vision + Piano System Roadmap

目标：重新定义完整产品方向，不急于上线，把钢琴系统纳入核心模块。

### P18 — Imported Target Practice Lite

目标：让本地导入目标从只读预览升级为可选择、可进入的轻量练习目标。

### P19 — Non-scoring Pitch Feedback

目标：让用户录音后得到非评分型反馈：

- 接近目标音
- 略偏高
- 略偏低
- 未检测到稳定音高
- 当前片段置信度较低，仅供参考

### P20 — Practice Piano Component Plan

目标：规划 `/practice` 内嵌钢琴组件。

### P21 — Practice Piano Component Runtime

目标：实现练习页可调用钢琴组件的最小版本。

### P22 — Standalone Piano Page Plan

目标：规划独立钢琴页面。

### P23 — Standalone Piano Page Runtime

目标：实现独立钢琴页面最小版本。

### P24 — Recording Management Plan

目标：规划录音管理。

### P25 — Piano Learning Mode Plan

目标：规划钢琴学习模式。

### P26 — Piano Accompaniment Plan

目标：规划钢琴伴奏系统。

### P27 — Complete MVP Release Gate QA

目标：只有核心功能基本真实实现后，才开始完整 release QA。
