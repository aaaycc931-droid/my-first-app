# Android 专业竞争力路线：钢琴、音准分析与视唱练耳系统

状态：**Canonical active roadmap / 已恢复开发的主路线**

决策日期：2026-07-16

当前执行状态：**ACTIVE / 产品所有者已于 2026-07-17 恢复运行时开发，当前推进 P106**

## 1. 产品决策

本项目不再把“参考钢琴能发声”“录音后能显示一个估计音高”或“四类听辨题可以作答”当作足够的产品竞争力。恢复开发后必须同时沿三条专业主轴推进：

1. 钢琴功能的功能广度、触控可靠性、低延迟和音色质量，以 Perfect Piano 一类成熟移动钢琴应用的核心演奏能力为参照；
2. 录音提交后的本地音准分析，以可重复数据集、多个候选引擎、音符分段、目标对齐、置信度和拒答能力建立专业证据；
3. 视唱、练耳、节奏、声乐练习、课程、定制和学习反馈，以 EarMaster 一类完整训练系统的活动覆盖、教学递进和交互闭环为参照。

“对标”是能力域和可验证质量门槛，不是复制竞品。不得复制其 UI、图标、音色文件、曲库、课程文案、题目、品牌表达或其他受保护资产。

## 2. 竞品事实基线与范围边界

本路线依据 2026-07-16 查阅的公开官方资料建立：

- [Perfect Piano Google Play 官方页面](https://play.google.com/store/apps/details?id=com.gamestar.perfectpiano)：公开列出 88 键、单排/双排/双人/和弦模式、多点触控、力度触控、键宽调整、多音色、MIDI/音频录音、节拍器、低延迟音频、学习模式、USB MIDI、音色插件和 MusicXML；
- [Revontulet Soft 官方页面](https://www.revontuletsoft.com/)：将 Perfect Piano 定位为移动钢琴模拟与学习工具；
- [EarMaster 官方完整功能页](https://www.earmaster.com/products/ear-training-sight-singing/earmaster-software.html)：公开列出音程、和弦/转位/进行、音阶、节奏视读/回模/找错/听写、旋律听写/回唱/视唱、麦克风与 MIDI 输入、课程、定制、统计和自适应题目；
- [EarMaster 官方移动版页面](https://www.earmaster.com/products/ear-training-sight-singing/earmaster-for-mobile.html)：公开列出移动端语音即时音高检测、节奏敲击、数千课节、采样乐器音色和自适应练习。

本项目吸收其核心专业能力，但继续明确不做：

- 社区、聊天室、公会、排行榜、多人竞技；
- 公开曲库、公开上传或公开分享用户素材；
- 复制竞品曲目、课程、题库、音色包或视觉设计；
- 未通过数据与教育审核的正式分数、等级或通过/失败；
- 当前 Android 本地版对账号、Supabase、Vercel 或远程 `server.url` 的启动依赖。

## 4. 专业钢琴系统目标

### 4.1 V1 专业基线

1. 完整 88 键逻辑音域 A0–C8；至少支持连续滚动与快速八度定位。
2. 单排、双排和练习内嵌紧凑模式；双人模式和和弦快捷模式在正式候选前完成或以明确降级状态关闭。
3. 多点触控至少 10 指状态正确；生产音频至少 32 音复音，目标 64 音。
4. 支持键宽、音名标签（科学音高/固定唱名/首调唱名）、中央 C、移调和横竖屏布局。
5. 支持可用时的触控压力/接触面积映射，并提供无压力能力时的稳定默认力度。
6. 默认使用合法授权、可追溯来源的采样三角钢琴；至少 3 个力度层、稀疏根音采样映射、自然衰减和无爆音释放。
7. 内置至少 6 类可辨识音色：三角钢琴、明亮钢琴、电钢琴、音乐盒、管风琴和合成音；默认核心包必须完全离线。
8. 延音踏板、半踏板能力检测或明确不支持提示、基础混响、主音量、立即全停和失焦/后台/卸载清理。
9. 内置节拍器、录制/停止/回放/重命名/删除/导出；录音至少保存可编辑事件序列，音频导出作为后续完整切片。
10. 支持 A–B 循环、速度调整、左右手/声部过滤，以及谱面、瀑布或落键中至少两种原创学习视图。
11. USB MIDI 输入、延音踏板、力度、通道和设备断开恢复；MIDI 不可用时触屏钢琴完整可用。
12. MusicXML/MXL 的已确认草稿可以进入钢琴学习模式；仍须遵守“预览 → 检查 → 修改/确认 → 再进入练习”。

### 4.2 音频工程门槛

- 音高调律误差不超过 ±2 cents；A4 基准支持 415–466 Hz，默认 440 Hz。
- 中档 Android 上用户手势到音频计划启动 P95 不高于 80 ms；真实扬声器/蓝牙输出延迟单独记录。
- 连续 10 指、32 音复音和延音释放不得出现可复现卡音、爆音、无界音量叠加或失控常驻声源。
- 默认钢琴包首次可弹时间 P95 不高于 2 秒；采用预加载关键根音、按需解码、LRU 缓存和内存上限。
- 低端设备资源不足时自动降级复音数、混响或力度层，但不得退回无提示的廉价单振荡器音色。
- 每个音色记录许可、来源、版本、采样率、归一化、循环点/非循环策略和构建摘要。

### 4.3 架构原则

```text
Piano UI / MIDI input
→ normalized note events (note-on/off, velocity, pedal, channel)
→ polyphonic voice allocator
→ versioned sample/timbre provider
→ envelope + pedal + effect bus
→ Android low-latency output
→ recorder / metronome / learning overlay
```

当前 Web Audio 三角波保留为显式“兼容降级音色”，不得继续作为默认专业钢琴。

## 5. 录音提交后的专业本地音准分析

### 5.1 与实时监听分离

实时监听优先低延迟，录音提交后的离线分析优先准确、稳定和可解释。两者必须共享标准音高帧协议，但使用独立算法版本和质量门槛：

```text
用户主动录音
→ 停止并确认分析
→ 本地解码为统一 PCM
→ DC/增益/噪声底诊断
→ 多候选音高与 voiced/unvoiced
→ 八度候选与连续轨迹优化
→ 音符/片段分段
→ 目标节拍与音符对齐
→ 逐音/逐句证据
→ 用户预览、检查、复练或删除
```

### 5.2 引擎路线

1. 保留当前 autocorrelation 作为可重复基线。
2. 将现有 comparison-only Pitchy/McLeod 适配为同帧评估候选，但未经基准不得直接替换生产引擎。
3. 增加 YIN/CMNDF 候选；评估 pYIN/HMM 或轻量 TFLite 单声部模型在三档 Android 上的收益与成本。
4. 使用 champion/challenger 协议比较 voiced F1、Raw Pitch Accuracy、Raw Chroma Accuracy、cents 误差、八度错误、延迟和实时系数。
5. 使用连续性、谐波证据和目标先验抑制八度跳变，但目标先验不得把错误演唱强行吸附成正确答案。
6. 无声、气声、强噪声、复音、伴奏泄漏和置信度不足必须拒答或局部拒答。

### 5.3 数据与验收

P104 现有门禁继续作为第一层最低证据：200 个合成/乐器片段、100 个真实人声片段、20 位参与者、4 类设备和 2 位教育审核者。专业对标报告还应逐步扩展到：

- 至少 1000 个可公开复现或有合法本地授权的合成/乐器测试片段；
- 至少 300 个真实人声片段、40 位参与者、6 类采集设备；
- 音域、元音、强弱、气声、颤音、滑音、尾音、噪声、扬声器泄漏和无音高反例分层；
- 调参集、开发验证集和最终盲测集严格隔离；
- 至少两名独立标注者；争议样本由第三人或教育审核者复核；
- 每次报告固定算法 commit、模型/参数版本、手机、Android、WebView/原生引擎、采样率和日期。

最低量化门槛继续采用正式 DoD：真实人声绝对音高误差中位数不高于 35 cents、P95 不高于 100 cents、八度错误率不高于 2%。在此基础上增加：

- voiced/unvoiced F1 不低于 0.95（分层报告，不用总体均值掩盖噪声/气声失败）；
- 清洁持续音 Raw Pitch Accuracy ±50 cents 不低于 97%；
- 低中高三档 Android 的离线处理实时系数 P95 不高于 0.5，即 10 秒录音最多约 5 秒完成；
- 相同输入、版本和参数必须产生确定性等价结果；
- 任何“正式分数”仍须另行完成教育校准、重复性、公平性和阈值审批。

### 5.4 用户反馈形式

第一阶段只提供逐音和逐句的可解释非评分反馈：目标音、检测音、cents 中位数/范围、音头/稳定段/尾音、节拍偏移、置信度、无法判断原因和一键分段复练。不得用单个总百分比掩盖缺帧、八度错误或节奏错位。

## 7. 课程、内容与学习系统

### 7.1 统一活动模型

所有题型必须复用版本化活动协议：`activityId`、技能标签、目标、难度、调号/拍号/谱号、音域、节拍、参考音色、答案模式、允许输入、解释、算法版本和内容版本。自动生成内容仍须经过题目验证器与教育抽样。

### 7.2 内容门槛

- 完成上述 14 个活动族，每个活动至少基础/进阶/挑战三档；
- 每档至少 40 个经验证的稳定题目或等价生成组合，不能只靠随机数宣称题量；
- 至少一条 120 课节的中文递进课程，覆盖乐理准备、听辨、节奏、视唱和声乐练习；
- 课程抽样正确率 100%，难度递进、唱名体系和中文解释由至少 2 名视唱练耳教师审核；
- 允许用户定制调号、谱号、音域、节拍、题目集合、声音、时限和答案界面；
- 记录按技能、题型、难度和时间的详细统计，并从错误项直接进入定向复练。

### 7.3 可解释自适应

自适应只使用版本化练习结果，在样本不足时不启用。推荐原因必须可见，例如“最近 10 次上行小六度识别错误 4 次”，用户可关闭建议、重置画像或手动选择练习。不得使用黑箱难度或把非评分音高观察写成能力等级。
