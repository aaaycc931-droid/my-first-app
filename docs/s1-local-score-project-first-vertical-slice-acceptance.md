# S1 本机基础谱项目首个垂直切片验收

## 定位

本切片落实 S1 的第一条可运行闭环：

> 新建一份谱 → 输入音符或休止 → 保存 → 播放 → 撤销/重做 → 返回列表并重新打开。

它是 S1 的运行时起点，不代表完整 S1、完整制谱器或正式推广版本完成。

## 本批范围

- 在 Android/Web 共用的移动端界面提供“本机谱项目”入口。
- 创建并立即保存一份带稳定项目 ID、文档 ID 和 revision 的本机谱。
- 编辑第一 part、第一 staff、第一 voice 的基础音符和休止事件。
- 支持当前受控音高、二分/四分/八分音符以及四分休止符。
- 每个编辑命令先通过 IndexedDB 原子持久化，成功后才发布到界面。
- 支持删除事件、撤销、重做，并把历史随项目一并保存。
- 主动播放当前谱；休止必须保留时间，不自动播放或打开麦克风。
- 返回项目列表后可重新打开同一项目，并恢复内容、revision 与撤销/重做历史。

## 必须通过的自动验收

1. 空列表可创建并保存项目；创建失败时不得进入一个看似已保存的编辑器。
2. 添加音符、添加休止、删除、撤销和重做均只产生一个连续 revision。
3. 写入失败或 stale writer 冲突时，界面继续显示最后已保存 revision，不得覆盖较新记录。
4. 非法事件位置、重复 ID、未知 schema、损坏记录和时钟回退必须 fail closed。
5. 单谱表、多谱表和多声部 fixture 继续确定性 round-trip；本 UI 不得删除未显示的其他声部数据。
6. 播放计划按文档 ID 与 revision 标识，限制 BPM，保留休止，同一时刻先 note-off，并在终点发送 all-notes-off。
7. 离开页面、应用进入后台、窗口失焦、全局停止或组件卸载时不得留下持续发声。
8. Quality 必须覆盖全部 `test:*` 脚本且每项恰好一次；lint、typecheck、构建和 Android 本地 APK 继续通过。

## 严格人工 QA

在真实浏览器和 Android WebView 上分别执行：

1. 创建项目，依次加入音符与休止，播放并主动停止。
2. 撤销、重做，返回列表，强制刷新或重启应用，再打开同一项目。
3. 确认内容、标题、revision 和可用历史与保存前一致。
4. 模拟两个页面编辑同一项目，确认后写入者收到冲突提示且不能静默覆盖。
5. 模拟存储不可用、空间不足或事务失败，确认未保存修改不出现在编辑器中。
6. 播放中切换页面、锁屏/进后台和恢复，确认声音停止且不会自行重播。
7. 断网完成以上流程，确认没有网络上传或账号依赖。
