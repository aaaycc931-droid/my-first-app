# 最终产品实现蓝图

> 规范关系：本文件定义实现架构原则。正式版 V1 的唯一范围、量化门槛和退出条件见 `docs/final-release-definition-of-done.md`，实时完成状态见 `docs/final-release-status-matrix.md`。

## 0. 迁移原则

现有 browser-local 练习能力必须继续可用，并逐步迁移，不以一次重写取代正在工作的练习流程。每个正式功能先建立数据契约、权限边界和删除语义，再接入 UI。不能在没有真实基准与教育验证前把诊断反馈改称正式评分。

Android 正式版采用 PWA + Trusted Web Activity，共用生产 Next.js 与 Supabase，不另建一套业务客户端。详细构建、签名、App Links、权限和真机门槛见 `docs/android-apk-release-plan.md`。后续每个核心功能必须在普通 Web 与已安装 Android APK 两条路径回归，不能把 Android 适配集中拖到项目末尾。

## 1. 数据契约

第一批正式实体：

- `profile`：用户资料、显示偏好、时区与删除状态；
- `course`、`lesson`、`exercise`：系统课程及其版本化目标；
- `practice_session`、`practice_attempt`：一次练习及其输入、状态和来源；
- `evaluation_result`：版本、置信度、指标、反馈和可复核原始摘要；
- `skill_progress`、`review_queue`：技能趋势和可解释的复习安排；
- `private_asset`、`analysis_job`、`notation_draft`：私有素材、后台处理与必须审核的草稿；
- `consent_record`、`deletion_request`、`audit_event`：授权、删除和安全审计。

所有用户拥有的数据表均以 `user_id` 为所有权主键，并由数据库行级安全策略限制为本人访问。练习目标、规则与评测结果必须保存 `version`，避免规则升级后篡改历史解释。

## 2. 认证、权限与私有数据

- 使用邮箱魔法链接和密码登录作为第一版认证；后续可添加受审查的第三方登录。
- 未登录用户只可使用明确标记为本地临时体验的功能；任何云端保存、上传、处理、报告与同步均要求登录。
- 对象存储桶为私有；上传和下载采用短期签名 URL；对象路径含用户所有者与资产 ID，禁止客户端任意路径写入。
- 提供“导出我的数据”和“删除我的数据”；删除先撤销访问，再删除对象、派生结果与数据库记录，并记录完成时间。

## 3. 评测与反馈路线

### 3.1 练习反馈

浏览器侧提供参考音、钢琴、节拍器、录音、实时辅助和即时诊断。用户主动开始录音，停止时释放媒体轨道；可在本地回放后决定是否上传用于深度分析。

### 3.2 云端深度分析

上传必须显示用途、文件大小/时长限制、保留期限和删除入口。worker 产生结构化输出：音高轨迹、起音、节拍对齐、置信度、异常和规则版本。评测 API 只保存可复核摘要；原始音频默认设置有限保留期。

### 3.3 正式评分门槛

正式分数必须有：真实设备录音集、标注规范、音高/节奏误差指标、置信度校准、不同声部/设备测试、音乐教育专家审核、规则版本和回归测试。门槛未完成时 UI 使用“练习反馈”“可靠性不足”“建议复练”，不使用通过、等级或考试分数。

## 4. 私有素材与识谱路线

1. MusicXML/MXL 为第一个正式结构化输入，生成可编辑草稿。
2. 图片/PDF 和音频为私有上传资产，进入异步任务，不在同步 API 或浏览器主线程执行重型处理。
3. OMR、音高/节奏/分句分析结果均带来源、模型/规则版本、置信度和不确定项。
4. 用户必须审核、编辑和确认草稿，才能生成可评测的正式练习目标。
5. 任务支持排队、运行、完成、失败、取消和重试；失败不丢失原始私有资产或已有草稿。

## 5. 正式发布验收

- 安全：行级安全测试、越权访问测试、签名 URL 时效、速率限制、依赖扫描。
- 数据：备份恢复演练、导出/删除端到端演练、迁移回滚策略。
- 音频：Chrome、Safari iOS、Android Chrome、常见麦克风/MIDI 设备测试。
- 质量：真实音频/乐谱基准、worker 失败注入、长任务取消、低网速恢复。
- 体验：中文主流程、键盘与屏幕阅读器、移动端、空状态、加载、错误、隐私说明。
- Android：签名 APK/AAB、Digital Asset Links、magic-link App Links、升级/回滚、三档真机上的音频/麦克风/录音/文件选择和断网恢复。

## 6. 外部接入清单

实施到相应阶段时需要部署环境提供：Supabase 项目 URL、匿名 key、服务端 service key、私有 Storage 桶、邮件发送域、后台 worker 运行环境、错误追踪 DSN，以及经许可的 OMR/音频分析运行环境。没有这些凭据时，代码仍可完成接口、迁移、模拟/本地测试和安全检查，但无法宣称云端能力已经上线。
